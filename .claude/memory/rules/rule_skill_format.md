# Skill 编写规范

> 所有 Skill 必须遵循此规范，确保通用性和可移植性。

## 核心原则

**Skill 是模板，不是任务记录。禁止出现任何项目特定的具体值。**

## 强制规则

| 规则 | 说明 |
|------|------|
| **禁止具体 ID** | 不能出现 `20260530_xxx`、`your-username` 等 |
| **禁止具体路径** | 不能用 `/home/user/xxx` 或 `.claude/memory/local/`（已废弃） |
| **禁止具体数值** | 不能用 `8/66`、`12%`，改用 `{已完成}/{总计}`、`{X}%` |
| **禁止硬编码 URL** | 不能用 `https://gitee.com/your-username/xxx`，改用 `{远程URL}` |

## 占位符规范

```
{任务ID}       — 任务标识符
{已完成}       — 已完成数量
{总计}         — 总数量
{X}%           — 百分比
{平台}         — Git 平台名（Gitee/GitHub）
{远程URL}      — 远程仓库地址
{默认分支}     — 主分支名
```

## 格式要求

1. Skill 文件头部必须包含格式规则声明
2. 所有示例用占位符，不用具体值
3. 新建 Skill 时，先检查是否违反此规范

## 示例

错误：
```
继续 20260530_项目实战_知识图谱内容扩展
```

正确：
```
继续 {任务ID}
```
