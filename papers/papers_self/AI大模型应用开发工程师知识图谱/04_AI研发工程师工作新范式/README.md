# 4. AI研发工程师工作新范式

> **本章定位**：工程组织与 AI Coding 方法论——面向「用 AI 写代码、带团队、做重构」的研发负责人与高级工程师。

## 学习目标

学完本章你应能：

- 说清「确定性驾驭概率性」在流水线、审查闭环中的体现
- 设计团队的 AI 流水线（上下文→生成→验证→合并）
- 制定大型遗留系统的 AI 重构策略与质量飞轮
- 描述 AI 时代团队分工（AI 指挥官、审查者、知识飞轮维护者）

## 建议学习顺序（约 2 周）

| 顺序 | 模块 | 核心议题 |
|:---:|------|----------|
| 1 | [4-1 范式变革](4-1_AI_Coding带来的范式变革/README.md) | 流水线、知识飞轮、审查闭环 |
| 2 | [4-2 大型项目](4-2_大型软件项目的AI开发与AI重构/README.md) | 架构思维、重构战略、质量飞轮 |
| 3 | [4-3 团队分工](4-3_AI_Coding中的团队重新分工与新协作模式/README.md) | 角色、流程、组织、文化 |

每篇建议配合 **Cursor Rules / CLAUDE.md** 在你真实仓库里试写一版规则（见 4-1-1、4-1-3）。
