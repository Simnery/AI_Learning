# 4.3.2 流程重构

> 4.3.2 流程重构 > 4.3 团队重新分工 > 4. AI研发工程师工作新范式

## 核心概念

- **是什么**：流程重构是把传统软件开发流程（需求→设计→开发→测试→部署）改造成适配 AI 协作的新流程。核心变化：AI 参与每个环节，人的角色从事务执行变为策略把控。
- **为什么重要**：旧流程是为"人写代码"设计的。把 AI 硬塞进去只是加速了局部环节，不改流程整体效率提升有限。流程重构后才能释放 AI 的 10 倍潜力。
- **前置知识**：理解 AI 指挥官角色（参考 [4-3-1 AI指挥官](4-3-1_从职能专家到AI指挥官.md)），了解审查闭环和质量飞轮。

### 新旧流程对比

```
旧流程:  需求 → 设计 → 编码 → 审查 → 测试 → 部署
         (人)   (人)   (人)   (人)   (人)   (人)

新流程:  需求 → AI写Spec → AI生成代码 → AI审查 → AI测试 → 部署
         (人确认) (人审核)  (人验收)    (人终审) (人抽查)  (人决策)
```

## 原理讲解

### 1. 构建团队的"生产性资本"

**生产性资本**是团队积累的、能持续产出价值的 AI 协作资产：

| 资产类型 | 内容 | 价值 |
|---------|------|------|
| Spec 库 | 已完成任务的标准化规格 | 同类任务直接复用，缩短启动时间 |
| Prompt 库 | 高频场景的 prompt 模板 | 新人也能产出高质量 AI 代码 |
| 规则库 | 编码规范 + 架构约束 | AI 产出自动符合团队标准 |
| 测试模板 | 按模块类型的测试框架 | 新模块自动生成全套测试 |
| 架构知识 | AI 可读的系统设计文档 | AI 能理解全局而非局部 |

### 2. 人机协同的"精准制导"

传统模式下人和 AI 是"一问一答"。精准制导模式下，人像导弹制导系统一样持续调整 AI 的方向：

```
精准制导三阶段:
┌─────────────┐   ┌─────────────┐   ┌─────────────┐
│  初始瞄准    │ → │  中途修正    │ → │  末端精确    │
│  给 Spec    │   │  给反馈     │   │  给微调     │
│  给上下文   │   │  指正偏差   │   │  给细节     │
└─────────────┘   └─────────────┘   └─────────────┘
```

### 3. 新流程的六个环节

| 环节 | 旧模式（人做） | 新模式（AI 做） | 人做什么 |
|------|-------------|---------------|---------|
| 需求分析 | 人理解需求 | AI 拆解需求→生成Spec | 确认和修正 |
| 方案设计 | 人写设计文档 | AI 生成多方案 | 选择方案 |
| 代码实现 | 人写代码 | AI 生成代码 | 审查验收 |
| 测试 | 人写测试 | AI 生成测试 | 补充探索性测试 |
| 审查 | 人逐行审查 | AI 预审→人终审 | 关注关键决策 |
| 文档 | 人写（或不写） | AI 自动生成 | 审核准确性 |

## 代码实战

### 示例1：流程自动化脚本

```python
# pip install — 本示例仅用 Python 3.10+ 标准库，无需额外安装

# 标准库示例，无需额外 pip 依赖
"""
新流程编排器: 自动化六环节的标准操作
"""

class NewFlowOrchestrator:
    """AI 时代的开发流程编排"""
    
    def __init__(self, llm_client, project_rules: str):
        self.llm = llm_client
        self.rules = project_rules
    
    def step1_analyze_requirement(self, raw_req: str) -> str:
        """需求分析 → AI 拆解为结构化 Spec"""
        prompt = f"""将以下需求拆解为结构化 Spec：
{raw_req}
{self.rules}

输出包含: 功能描述/输入输出/边界条件/依赖/验收标准"""
        return self.llm.complete(prompt)
    
    def step2_design_solutions(self, spec: str) -> list[str]:
        """方案设计 → AI 生成多个备选方案"""
        prompt = f"""基于 Spec 生成 3 个实现方案：
{spec}

每个方案含: 思路/优点/缺点/适用场景"""
        return self.llm.complete(prompt).split("\n---\n")
    
    def step3_generate_code(self, spec: str, chosen_solution: str) -> str:
        """代码实现 → AI 按方案生成代码"""
        prompt = f"""实现以下 Spec，按选定方案：
{spec}
方案: {chosen_solution}
{self.rules}"""
        return self.llm.complete(prompt)
    
    def step4_generate_tests(self, code: str, spec: str) -> str:
        """测试 → AI 生成测试"""
        prompt = f"""为以下代码生成 pytest 测试：
代码: {code}
Spec: {spec}
覆盖: 正常/边界/异常"""
        return self.llm.complete(prompt)
    
    def step5_ai_review(self, code: str, tests: str) -> str:
        """审查 → AI 预审"""
        prompt = f"""审查代码: {code}
测试: {tests}
输出审查报告: 安全/性能/可维护性"""
        return self.llm.complete(prompt)
    
    def step6_generate_docs(self, code: str, spec: str) -> str:
        """文档 → AI 生成"""
        prompt = f"""为此功能生成文档: 概述/API/使用示例
代码: {code}
Spec: {spec}"""
        return self.llm.complete(prompt)
    
    def run(self, requirement: str) -> dict:
        """运行完整新流程"""
        print("1/6 需求分析...")
        spec = self.step1_analyze_requirement(requirement)
        
        print("2/6 方案设计...")
        solutions = self.step2_design_solutions(spec)
        
        # 人选方案
        chosen = solutions[0]  # 简化：选第一个
        print(f"   选定方案 ({len(solutions)} 个可选)")
        
        print("3/6 代码生成...")
        code = self.step3_generate_code(spec, chosen)
        
        print("4/6 测试生成...")
        tests = self.step4_generate_tests(code, spec)
        
        print("5/6 AI 审查...")
        review = self.step5_ai_review(code, tests)
        
        print("6/6 文档生成...")
        docs = self.step6_generate_docs(code, spec)
        
        return {"spec": spec, "code": code, "tests": tests, "review": review, "docs": docs}
```

### 示例2：团队生产性资本管理

```python
"""
生产性资本管理器: 团队 AI 资产的版本化管理
"""

class ProductiveCapital:
    def __init__(self, base_path: str = ".claude"):
        self.base = Path(base_path)
        self.specs_dir = self.base / "specs"
        self.prompts_dir = self.base / "prompts"
        self.rules_dir = self.base / "rules"
        
        for d in [self.specs_dir, self.prompts_dir, self.rules_dir]:
            d.mkdir(parents=True, exist_ok=True)
    
    def add_spec(self, feature_name: str, spec: str):
        name = feature_name.replace(" ", "_").lower()
        (self.specs_dir / f"{name}.md").write_text(spec)
    
    def find_similar_spec(self, requirement: str) -> str:
        """查找相似 Spec（简化版关键词匹配）"""
        for f in self.specs_dir.glob("*.md"):
            content = f.read_text()
            words = set(requirement.lower().split())
            score = sum(1 for w in words if w in content.lower())
            if score > 3:
                return content
        return ""
    
    def get_capital_report(self) -> dict:
        """生产性资本盘点"""
        return {
            "specs": len(list(self.specs_dir.glob("*.md"))),
            "prompts": len(list(self.prompts_dir.glob("*.md"))),
            "rules": len(list(self.rules_dir.glob("*.md"))),
        }
```

## 常见问题

### 自检题

**Q1**：什么是团队的"生产性资本"？包含哪些资产？

<details><summary>答案</summary>

生产性资本是团队积累的 AI 协作资产：Spec 库（标准化任务规格）、Prompt 库（高频场景模板）、规则库（编码/架构约束）、测试模板、架构知识文档。这些资产让 AI 产出质量和团队效率持续提升。
</details>

**Q2**：新流程六环节中，人的角色从"执行者"变成了什么？

<details><summary>答案</summary>

需求分析 → 确认和修正 Spec；方案设计 → 选择方案；代码实现 → 审查验收；测试 → 补充探索性测试；审查 → 关注关键决策；文档 → 审核准确性。人变成了"审核者+决策者"。
</details>

**Q3**："精准制导"三个阶段的含义是什么？

<details><summary>答案</summary>

初始瞄准（给 Spec + 上下文）→ 中途修正（给反馈 + 指正偏差）→ 末端精确（给微调 + 细节）。人持续引导 AI，而不是一次性提问。类比导弹需要持续制导才能命中目标。
</details>
