# 4.3.4 文化重塑

> 4.3.4 文化重塑 > 4.3 团队重新分工 > 4. AI研发工程师工作新范式

## 核心概念

- **是什么**：文化重塑是 AI 编程时代最深层的变革——改变团队对"工作"的根本认知。从"写代码 = 生产力"变成"决策质量 = 生产力"，从"AI 是工具"变成"AI 是团队成员"，从"追求完美第一版"变成"追求快速迭代"。
- **为什么重要**：流程可以改、组织可以调，但如果文化不变，一切新制度都会在旧文化的土壤上枯萎。AI 的 10 倍效率需要 10 倍不同的思维模式来承载。
- **前置知识**：理解流程重构（参考 [4-3-2 流程重构](4-3-2_流程重构.md)）和组织进化（参考 [4-3-3 组织进化](4-3-3_组织进化.md)）。

## 原理讲解

### 1. 六项文化转变

| # | 从 | 到 | 为什么 |
|---|-----|----|--------|
| 1 | **代码行数 = 产出** | **正确决策 + 质量 = 产出** | AI 写得比人快，比行数没意义 |
| 2 | **AI 是工具** | **AI 是初级团队成员** | 工具不需要 Spec 和管理，团队成员需要 |
| 3 | **人写代码，AI 配合** | **AI 写代码，人审查指挥** | 角色反转，人做高价值判断 |
| 4 | **追求完美第一版** | **追求快速迭代** | AI 能 30 秒生成新版，不需要一次到位 |
| 5 | **个人英雄主义** | **团队飞轮主义** | 一个人的 prompt 好不够，全团队的 prompt 都好才行 |
| 6 | **怕犯错** | **怕不迭代** | AI 时代最大的错误是不利用 AI，而不是 AI 产出有小问题 |

### 2. 文化落地的三个锚点

```
锚点1: 语言
  "AI 帮我写的" → "我指挥 AI 实现的"
  改变话语体系 = 改变思维模式

锚点2: 仪式
  每周 "Prompt Show": 分享本周最好用的 prompt
  每月 "AI Retro": 回顾 AI 协作中的问题和改进
  每季度 "飞轮审计": 审查规则库和 Spec 库的健康度

锚点3: 激励
  奖励: 分享高质量 prompt/规则的人
  奖励: 发现并修复 AI 系统性问题的质量验证师
  不奖励: 纯粹靠堆 AI 代码量完成的功能
```

### 3. 新人的 AI 文化融入

```
第 1 周: "看" —— 观察团队如何和 AI 协作
   - 参加 Prompt Show
   - 读团队的 Spec 库和规则库
   - 看队友的 AI 交互记录

第 2 周: "跟" —— 在指挥官指导下和 AI 协作
   - 领一个任务，在 AI 指挥官指导下完成
   - Spec 由指挥官写，新人负责把 Spec 喂给 AI 并验证产出

第 3 周: "做" —— 独立完成任务
   - 自主写 Spec → 指挥 AI → 验收 → 提交
   - 每周至少分享 1 个有效 prompt

第 4 周: "教" —— 帮助别人
   - 参与审查队友的 AI 产出
   - 贡献至少 1 条规则/1 个 prompt 到团队库
```

### 4. AI 文化健康度指标

| 指标 | 健康信号 | 危险信号 |
|------|---------|---------|
| Prompt 分享率 | 每周 > 2 个新 prompt 入库 | 一个月没新 prompt |
| AI 产出率 | > 60% 代码由 AI 初版生成 | < 20%（还在手写） |
| Spec 复用率 | > 30% 新任务复用已有 Spec | 每次都从头写 |
| 迭代轮次 | 核心功能 3-5 轮飞轮迭代 | 1 轮就合入，5+ 轮还在微调 |
| 知识沉淀 | 每两周规则库有更新 | 规则库超过 1 个月未更新 |

## 代码实战

### 示例1：文化健康度仪表盘

```python
# pip install — 本示例仅用 Python 3.10+ 标准库，无需额外安装

# 标准库示例，无需额外 pip 依赖
"""
AI 文化健康度监控: 量化追踪团队文化转型进度
"""

class CultureHealthDashboard:
    def __init__(self, claude_dir: str = ".claude"):
        self.claude = Path(claude_dir)
    
    def check_prompt_health(self, days: int = 14) -> dict:
        """检查 Prompt 库活跃度"""
        prompt_dir = self.claude / "prompts"
        if not prompt_dir.exists():
            return {"status": "critical", "message": "Prompt 库不存在！"}
        
        recent = []
        cutoff = datetime.now() - timedelta(days=days)
        for f in prompt_dir.glob("*.md"):
            mtime = datetime.fromtimestamp(f.stat().st_mtime)
            if mtime > cutoff:
                recent.append(f.stem)
        
        return {
            "status": "healthy" if len(recent) >= 2 else "warning",
            "recent_count": len(recent),
            "recent_prompts": recent
        }
    
    def check_rules_health(self, days: int = 30) -> dict:
        """检查规则库活跃度"""
        rules_dir = self.claude / "rules"
        if not rules_dir.exists() or not list(rules_dir.glob("*.md")):
            return {"status": "critical", "message": "规则库为空！"}
        
        # 找最旧的规则
        oldest = None
        oldest_date = datetime.now()
        for f in rules_dir.glob("*.md"):
            mtime = datetime.fromtimestamp(f.stat().st_mtime)
            if mtime < oldest_date:
                oldest_date = mtime
                oldest = f.stem
        
        days_old = (datetime.now() - oldest_date).days
        return {
            "status": "healthy" if days_old <= days else "warning",
            "oldest_rule": oldest,
            "days_since_update": days_old
        }
    
    def generate_report(self) -> str:
        """生成文化健康度报告"""
        prompt = self.check_prompt_health()
        rules = self.check_rules_health()
        
        report = [
            "# AI 文化健康度周报",
            f"## Prompt 库: {prompt['status']}",
            f"  近两周新增: {prompt['recent_count']} 个",
            f"## 规则库: {rules['status']}",
            f"  最久未更新: {rules.get('oldest_rule', 'N/A')} ({rules.get('days_since_update', 'N/A')} 天)",
            "\n## 行动建议"
        ]
        
        if prompt['status'] != 'healthy':
            report.append("- ⚠️  Prompt 分享不足，建议本周五开 Prompt Show")
        if rules['status'] != 'healthy':
            report.append("- ⚠️  规则库过时，建议安排飞轮审计")
        
        return "\n".join(report)
```

### 示例2：新人 AI 融入引导器

```python
"""
新人 AI 文化融入助手: 自动化四周融入计划
"""

class OnboardingGuide:
    WEEKLY_PLAN = {
        1: {
            "theme": "看",
            "tasks": [
                "阅读团队 CLAUDE.md 和规则库",
                "参加一次 Prompt Show",
                "浏览 Spec 库（至少 5 个已完成 Spec）",
                "看 3 段队友的 AI 交互记录"
            ]
        },
        2: {
            "theme": "跟",
            "tasks": [
                "在指挥官指导下完成 2 个任务",
                "学习如何使用团队 prompt 模板",
                "练习写第一个 Spec"
            ]
        },
        3: {
            "theme": "做",
            "tasks": [
                "独立完成 3 个任务（Spec → AI → 验收）",
                "分享 1 个有效 prompt 到团队库",
                "提交 1 条规则改进建议"
            ]
        },
        4: {
            "theme": "教",
            "tasks": [
                "参与审查队友的 AI 产出",
                "贡献 1 个 prompt 模板",
                "写一篇 AI 协作心得"
            ]
        }
    }
    
    def get_week_plan(self, week: int) -> dict:
        return self.WEEKLY_PLAN.get(week, {"theme": "？", "tasks": []})
    
    def check_progress(self, week: int, completed: list[str]) -> dict:
        plan = self.WEEKLY_PLAN[week]
        done = len(completed)
        total = len(plan["tasks"])
        return {
            "week": week,
            "theme": plan["theme"],
            "progress": f"{done}/{total}",
            "ready_for_next": done == total
        }
```

## 常见问题

### 自检题

**Q1**：AI 文化重塑的六项核心转变是什么？

<details><summary>答案</summary>

1. 代码行数→正确决策+质量。2. AI是工具→AI是团队成员。3. 人写代码→AI写代码人审查。4. 追求完美第一版→快速迭代。5. 个人英雄→团队飞轮。6. 怕犯错→怕不迭代。
</details>

**Q2**：文化落地的三个锚点是什么？

<details><summary>答案</summary>

语言（改变话语体系："AI帮我写"→"我指挥AI实现"）、仪式（每周Prompt Show、每月AI Retro、每季度飞轮审计）、激励（奖励分享prompt/规则和发现系统问题的人，不奖励纯堆AI代码量）。
</details>

**Q3**：AI 文化健康度的五个指标是什么？

<details><summary>答案</summary>

Prompt 分享率（每周新 prompt 数）、AI 产出率（AI 初版代码占比）、Spec 复用率（新任务复用已有 Spec 的比例）、迭代轮次（核心功能飞轮迭代次数）、知识沉淀（规则库更新频率）。
</details>
