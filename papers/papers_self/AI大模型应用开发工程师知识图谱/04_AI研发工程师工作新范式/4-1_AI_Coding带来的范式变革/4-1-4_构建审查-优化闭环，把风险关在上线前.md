# 4.1.4 构建"审查-优化"闭环，把风险关在上线前

> 4.1.4 构建审查-优化闭环 > 4.1 AI Coding带来的范式变革 > 4. AI研发工程师工作新范式

## 核心概念

- **是什么**：审查-优化闭环是 AI 编码时代的质量保障体系——AI 生成代码后，通过"自动审查 → 修复 → 再审查"的循环，确保只有高质量代码才能上线。人的最终决策权是闭环的最后且最关键的环节。
- **为什么重要**：AI 编码速度是人手的 10 倍，但产生的 bug 也可能是 10 倍。没有闭环的 AI 编程 = 往生产环境扔定时炸弹。闭环让你享受 AI 的速度，同时守住质量底线。
- **前置知识**：理解 AI 流水线（参考 [4-1-2 构建AI流水线](4-1-2_构建你的AI流水线.md)），了解知识飞轮（参考 [4-1-3 知识飞轮](4-1-3_构建团队的知识飞轮.md)）。

### 闭环公式

```
AI 速度 × 审查闭环 = 可控的高质量交付
AI 速度 × 无审查    = 灾难加速器
```

## 原理讲解

### 1. 审查闭环的四层防线

```
┌─────────────────────────────────────────────┐
│  第 4 层: 人工终审                          │
│  人检查 AI 审查结果 + 做最终决策              │  ← 不可绕过
├─────────────────────────────────────────────┤
│  第 3 层: AI 深度审查                        │
│  安全漏洞 / 业务逻辑错误 / 性能问题           │
├─────────────────────────────────────────────┤
│  第 2 层: 自动化测试                          │
│  单元测试 + 集成测试 + E2E 测试               │
├─────────────────────────────────────────────┤
│  第 1 层: 静态检查                            │
│  Lint + Type Check + 格式化                   │  ← 最快，秒级反馈
└─────────────────────────────────────────────┘
```

### 2. 每一层的职责和工具

| 层级 | 检查什么 | 工具 | 反馈速度 |
|------|---------|------|---------|
| L1 静态 | 语法/类型/格式 | ruff, mypy, biome | 秒级 |
| L2 测试 | 功能正确性 | pytest, vitest | 分钟级 |
| L3 AI审查 | 安全/逻辑/设计 | Claude Code, CodeRabbit | 分钟级 |
| L4 人工 | 业务判断/取舍 | 人眼 + 大脑 | 可控 |

### 3. 从"超级个体"到"超级团队"

AI 编程时代的一个常见陷阱：一个人 + AI 疯狂输出代码 → 成为"超级个体" → 但代码质量参差不齐，其他人看不懂也接不住。

**正确的演进路径**：

```
超级个体（危险）                   超级团队（健康）
     │                                  │
一个人 + AI 狂写             每个人 + AI 在自己领域高效产出
     │                                  │
代码只有自己懂               审查闭环保证质量统一
     │                                  │
离职 = 项目死亡              知识飞轮保证经验传承
     │                                  │
不可持续                     可持续
```

**关键转变**：AI 编码能力的 10 倍提升，必须配合团队协作能力的同步提升。

### 4. 人的最终决策权——AI 不能替代的判断

以下决策必须由人做出，AI 只能提供建议：

| 决策类型 | 为什么不能交给 AI |
|----------|-------------------|
| 架构选型 | AI 不了解团队能力和长期维护成本 |
| 安全策略 | AI 可能忽略特定业务的合规要求 |
| 用户体验 | AI 没有"人类直觉"，不懂情感设计 |
| 技术债务取舍 | AI 无法判断"现在还债"还是"以后还"的商业影响 |
| 发布决策 | AI 不了解业务压力和风险偏好 |

### 示例1：四层审查自动化脚本

```python
# pip install — 本示例仅用 Python 3.10+ 标准库，无需额外安装

# 标准库示例，无需额外 pip 依赖
"""
pre-commit-review.py —— 提交前的四层自动化审查
用法: python pre-commit-review.py --files changed_files.txt
"""

import subprocess
import sys
from pathlib import Path

class FourLayerReview:
    def __init__(self, changed_files: list[str]):
        self.files = changed_files
    
    def layer1_static_check(self) -> bool:
        """第1层: 静态检查"""
        print("=" * 40)
        print("第1层: 静态检查")
        
        all_passed = True
        
        # Python: ruff 检查
        py_files = [f for f in self.files if f.endswith('.py')]
        if py_files:
            r = subprocess.run(["ruff", "check", "--fix"] + py_files, capture_output=True, text=True)
            if r.returncode != 0:
                print(f"❌ Ruff 发现问题:\n{r.stdout}")
                all_passed = False
            else:
                print(f"✅ Ruff 通过 ({len(py_files)} 文件)")
        
        # TypeScript/React: biome 检查
        ts_files = [f for f in self.files if f.endswith(('.ts', '.tsx'))]
        if ts_files:
            r = subprocess.run(["npx", "biome", "check"] + ts_files, capture_output=True, text=True)
            if r.returncode != 0:
                print(f"❌ Biome 发现问题")
                all_passed = False
            else:
                print(f"✅ Biome 通过 ({len(ts_files)} 文件)")
        
        return all_passed
    
    def layer2_auto_test(self) -> bool:
        """第2层: 自动化测试"""
        print("\n" + "=" * 40)
        print("第2层: 自动化测试")
        
        r = subprocess.run(
            ["pytest", "-x", "--tb=short", "-q"],
            capture_output=True, text=True, timeout=120
        )
        if r.returncode == 0:
            print("✅ 全部测试通过")
            return True
        else:
            print(f"❌ 测试失败:\n{r.stdout[-500:]}")  # 只显示最后500字
            return False
    
    def layer3_ai_review(self) -> tuple[bool, str]:
        """第3层: AI 深度审查"""
        print("\n" + "=" * 40)
        print("第3层: AI 代码审查")
        
        # 构建审查 prompt（调用 AI）
        diff = self._get_git_diff()
        review_prompt = f"""审查以下 git diff，重点关注:

1. 🔴 安全: SQL注入、XSS、密钥硬编码、路径遍历
2. 🔴 正确性: 逻辑错误、边界条件遗漏、空指针
3. 🟡 性能: N+1查询、大循环中的低效操作
4. 🟡 可维护性: 过度复杂、魔法数字、不一致的命名

Git Diff:
{diff}

输出格式:
SEVERITY: <critical|warning|suggestion>
FILE: <文件路径>
LINE: <行号>
ISSUE: <问题描述>
FIX: <修复建议>
---
"""
        # 这里调用 AI（伪代码）
        # ai_response = llm.complete(review_prompt)
        ai_response = "SEVERITY: suggestion\nFILE: example.py\nLINE: 10\nISSUE: 使用可变默认参数\nFIX: 改用 None + 内部初始化"
        
        # 解析 AI 响应
        has_critical = "SEVERITY: critical" in ai_response
        return not has_critical, ai_response
    
    def layer4_human_check(self, ai_review: str) -> bool:
        """第4层: 人工终审"""
        print("\n" + "=" * 40)
        print("第4层: 人工终审")
        print(f"\nAI 审查结果:\n{ai_review}")
        print("\n请检查以上内容。")
        print("特别关注标记为 'critical' 和 'warning' 的项。")
        
        response = input("\n是否通过人工审查? [y/N]: ").strip().lower()
        return response == 'y'
    
    def _get_git_diff(self) -> str:
        """获取当前变更的 diff"""
        r = subprocess.run(
            ["git", "diff", "--cached"] + self.files,
            capture_output=True, text=True
        )
        diff = r.stdout
        if len(diff) > 15000:  # 限制长度
            diff = diff[:15000] + "\n... (diff 过长，已截断)"
        return diff
    
    def run(self) -> bool:
        """运行完整四层审查"""
        print("🚀 开始四层审查流程\n")
        
        # 第1层: 秒级
        if not self.layer1_static_check():
            print("\n⛔ 第1层未通过，请修复后重试")
            return False
        
        # 第2层: 分钟级
        if not self.layer2_auto_test():
            print("\n⛔ 第2层未通过，请先修复测试")
            return False
        
        # 第3层: AI审查
        l3_passed, ai_review = self.layer3_ai_review()
        if not l3_passed:
            print("\n⚠️  AI 审查发现严重问题，建议修复后继续")
            # 不阻断（有严重问题时仍然可以人工决定继续）
        
        # 第4层: 人工
        if not self.layer4_human_check(ai_review):
            print("\n⛔ 人工审查未通过")
            return False
        
        print("\n✅ 四层审查全部通过! 可以合并/部署。")
        return True

# 使用
# review = FourLayerReview(["src/main.py", "src/utils.py"])
# review.run()
```

### 示例2：AI 审查 + 自动修复循环

```python
"""
AutoFix Pipeline: AI 审查发现问题 → 自动修复 → 再审查
"""

class AutoFixPipeline:
    def __init__(self, llm_client, max_iterations: int = 3):
        self.llm = llm_client
        self.max_iterations = max_iterations
    
    def review_code(self, code: str) -> list[dict]:
        """AI 审查代码，返回问题列表"""
        prompt = f"""审查以下代码，找出所有问题。

代码:
{code}

输出 JSON 数组，每个问题格式:
{{"severity": "critical|warning|suggestion",
  "line": <行号>,
  "description": "问题描述",
  "fix_suggestion": "具体的修复建议"}}

只输出 JSON，不要其他。"""
        
        resp = self.llm.complete(prompt)
        import json
        return json.loads(resp)
    
    def fix_issues(self, code: str, issues: list[dict]) -> str:
        """根据审查意见修复代码"""
        issues_text = "\n".join(
            f"- [{i['severity']}] 第{i['line']}行: {i['description']}\n  建议: {i['fix_suggestion']}"
            for i in issues
        )
        
        prompt = f"""请修复以下代码中的所有问题。

当前代码:
{code}

需要修复的问题:
{issues_text}

请输出完整的修复后代码，不要省略任何部分。"""

        return self.llm.complete(prompt)
    
    def run(self, code: str) -> tuple[str, list[dict]]:
        """运行自动修复循环"""
        current_code = code
        all_fixed = []
        
        for iteration in range(self.max_iterations):
            print(f"\n--- 第 {iteration + 1} 轮 ---")
            
            # 审查
            issues = self.review_code(current_code)
            critical = [i for i in issues if i['severity'] == 'critical']
            warnings = [i for i in issues if i['severity'] == 'warning']
            
            print(f"发现问题: {len(critical)} 严重, {len(warnings)} 警告, {len(issues)-len(critical)-len(warnings)} 建议")
            
            if not critical and not warnings:
                print("✅ 无严重问题，审查通过!")
                return current_code, all_fixed
            
            # 只自动修复 critical 和 warning
            to_fix = critical + warnings
            print(f"自动修复 {len(to_fix)} 个问题...")
            
            current_code = self.fix_issues(current_code, to_fix)
            all_fixed.extend(to_fix)
        
        print(f"\n⚠️  达到最大迭代次数 {self.max_iterations}，剩余问题需人工处理")
        return current_code, all_fixed
```

```yaml
# .github/ai-contribution-guide.yml
# AI 贡献代码的团队规范

ai_contribution_rules:
  # 1. AI 生成的代码必须标记
  attribution:
    required: true
    format: "Co-Authored-By: {AI_Tool} <{version}>"
    example: "Co-Authored-By: Claude Code <noreply@anthropic.com>"
  
  # 2. 必须通过的最低审查层级
  min_review_level: "L3"  # 至少通过 AI 审查
  
  # 3. 禁止 AI 直接操作的生产环境
  forbidden_direct_operations:
    - "production database migration"
    - "secrets rotation"
    - "payment/billing code changes"
    - "auth system modifications"
  
  # 4. PR 描述必须包含
  required_pr_sections:
    - "what: 改了什么"
    - "why: 为什么改"
    - "ai_involvement: AI 参与了哪些部分"
    - "human_review: 人审查了什么"
    - "risk_assessment: 风险评估（低/中/高）"
  
  # 5. 测试覆盖率要求（AI 生成代码更高）
  coverage_threshold:
    human_written: 70
    ai_generated: 85  # AI 代码要求更高覆盖率
```

## 进阶方向

- 四层审查增加了流程开销，小改动可能不值得全跑
- AI 审查的误报率需要持续调优（否则变成"狼来了"效应）
- 自动修复可能引入新问题，过度迭代会降低信任

## 常见问题

### 小白最常踩的坑

1. **跳过 L1/L2 直接上 AI 审查**：AI 不应该被用来检查"少了个分号"。用 L1 先扫掉低级问题，AI 专注安全/逻辑
2. **把 AI 审查当终极真理**：AI 审查也可能误报漏报。永远需要人的判断作为最后防线
3. **审查太严格导致效率暴跌**：根据项目阶段调整标准，原型期不要用生产期的标准

### 自检题

**Q1**：审查闭环的四层防线分别是什么？为什么越底层越自动化？

<details><summary>答案</summary>

L1 静态检查（语法/格式）→ L2 自动测试（功能正确性）→ L3 AI 审查（安全/逻辑/设计）→ L4 人工终审（业务判断）。越底层的问题越确定（语法错误就是错误），越自动化；越上层越需要人类判断（安全策略可能有例外），越需要人工参与。
</details>

**Q2**："超级个体"的危险是什么？如何转向"超级团队"？

<details><summary>答案</summary>

超级个体：一个人 + AI 狂写代码，效率高但质量参差、知识孤岛、离职即项目死。转向超级团队：建立审查闭环保证质量统一、知识飞轮保证经验传承、明确的人机分工保证可协作。
</details>

**Q3**：哪些决策场景绝不能完全交给 AI？为什么？

<details><summary>答案</summary>

- 架构选型：AI 不了解团队能力和维护成本
- 安全策略：AI 可能忽略特定合规要求
- 用户体验：AI 缺乏人类直觉和情感理解
- 技术债务取舍：AI 无法评估商业影响
- 发布决策：AI 不了解业务压力和风险偏好
</details>
