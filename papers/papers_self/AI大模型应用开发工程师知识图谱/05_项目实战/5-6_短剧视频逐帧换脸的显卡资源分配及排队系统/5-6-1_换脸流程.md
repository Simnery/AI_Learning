# 5.6.1 换脸流程

> 5.6.1 换脸流程 > 5.6 短剧视频换脸 > 5. 项目实战

## 核心概念

- **是什么**：短剧视频逐帧换脸是一个 GPU 密集型计算任务——输入一段视频和一张目标人脸，逐帧检测人脸→提取特征→替换→合成输出。核心挑战不是算法本身，而是海量帧的高效处理和算力调度。
- **为什么重要**：换脸项目的技术栈涵盖视频处理、人脸检测、GPU计算、任务队列、资源管理——是一个完整的 AI 工程化案例。

## 原理讲解

### 换脸流水线

```
视频上传 → 帧提取(ffmpeg) → 人脸检测 → 人脸对齐 → 特征提取 → 换脸推理 → 帧合成 → 视频输出
              ↓                   ↓           ↓
         GPU解码            标注人脸框    关键点检测
```

### 任务生命周期

```
PENDING → QUEUED → PROCESSING → COMPLETED
   ↓                    ↓           ↓
CANCELLED            FAILED → RETRYING
```


### 3. 短剧换脸端到端流程

```
视频拆帧 → 人脸检测(5-6-3) → 逐帧换脸推理 → 融合回写 → 编码输出
     ↑                              ↑
  任务入队(5-6-2)              GPU 池调度
```

**瓶颈**：推理占 70%+ 时间；检测可 CPU/GPU 混部。

### 4. 质量检查点

| 指标 | 目标 |
|------|------|
| 人脸检测召回 | >98% 正脸帧 |
| 融合边界色差 | 目视无硬边 |
| 时序闪烁 | 相邻帧 ID 一致 |

## 代码实战

```python
# pip install — 本示例仅用 Python 3.10+ 标准库

"""换脸任务管理"""
from enum import Enum
import uuid, asyncio

class TaskStatus(Enum):
    PENDING = "pending"
    QUEUED = "queued"  
    PROCESSING = "processing"
    COMPLETED = "completed"
    FAILED = "failed"
    CANCELLED = "cancelled"

class FaceSwapTask:
    def __init__(self, video_path: str, face_image: str, params: dict = None):
        self.id = str(uuid.uuid4())
        self.video_path = video_path
        self.face_image = face_image
        self.params = params or {}
        self.status = TaskStatus.PENDING
        self.progress = 0.0  # 0-1
        self.result_path = None
        self.error_msg = None
    
    def to_dict(self):
        return {"id": self.id, "status": self.status.value, 
                "progress": self.progress, "result": self.result_path}

class FaceSwapPipeline:
    def __init__(self, face_detector, face_swapper, frame_extractor):
        self.detector = face_detector
        self.swapper = face_swapper
        self.extractor = frame_extractor
    
    async def process(self, task: FaceSwapTask):
        task.status = TaskStatus.PROCESSING
        
        # 1. 提取视频帧
        frames = await self.extractor.extract(task.video_path)
        total = len(frames)
        
        # 2. 逐帧处理
        results = []
        for i, frame in enumerate(frames):
            faces = self.detector.detect(frame)  # 人脸检测
            if faces:
                swapped = self.swapper.swap(frame, faces[0], task.face_image)
                results.append(swapped)
            else:
                results.append(frame)  # 无脸帧保持原样
            task.progress = (i + 1) / total
        
        # 3. 合成输出视频
        task.result_path = await self.extractor.compose(results)
        task.status = TaskStatus.COMPLETED
```

## 进阶方向

### 当前方案的局限性

- **逐帧串行**：虽然每帧独立可并行，但当前实现是逐帧串行处理，没有充分利用 GPU 批处理能力
- **人脸检测精度**：侧脸、遮挡、暗光场景下检测率会明显下降，无脸帧直接跳过可能遗漏可修复帧
- **换脸一致性**：逐帧独立处理可能导致相邻帧之间出现抖动——脸的位置/大小在两帧之间微小偏移
- **视频质量损失**：重复编解码（提取→处理→合成）导致画质下降

### 工业界最佳实践

- **帧间平滑**：相邻帧的换脸结果做时序平滑（光流法或滑动平均），消除抖动
- **自适应跳帧**：变化小的帧（静态场景）降低处理频率，节省 GPU 算力
- **断点续传**：支持从中间帧重新开始，处理中断后不用全重来

## 常见问题

### 小白最常踩的 3 个坑

**坑 1：视频编码格式导致帧提取失败**

某些视频用 H.265/HEVC 编码，ffmpeg 提取帧时需要额外解码器。处理前用 `ffprobe` 检查视频编码格式，H.265 需安装对应 codec。

**坑 2：人脸检测阈值设太高，侧脸全漏**

默认检测阈值 0.9，侧脸和暗光场景会大量漏检。换脸场景可以适当降低阈值到 0.6-0.7，宁可多检（假阳性可以后处理过滤），不要漏检。

**坑 3：逐帧处理后音画不同步**

换脸只改了画面没动音频，但帧处理时间不一致会导致音画偏移。输出时必须用原视频的帧率（fps）和时间基准重新合成。

### 自检题

**Q1**：换脸流水线包含哪些核心步骤？<details><summary>答案</summary>视频帧提取→人脸检测→人脸对齐→特征提取→换脸推理→帧合成→视频输出。每帧独立处理，天然适合并行。</details>

**Q2**：任务状态有哪些？失败后如何重试？<details><summary>答案</summary>PENDING→QUEUED→PROCESSING→COMPLETED/FAILED/CANCELLED。失败后可设置自动重试（RETRYING），最多重试3次。</details>
