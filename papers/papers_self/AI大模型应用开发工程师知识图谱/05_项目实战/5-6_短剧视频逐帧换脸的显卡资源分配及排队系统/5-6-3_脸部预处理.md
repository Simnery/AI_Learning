# 5.6.3 脸部预处理

> 5.6.3 脸部预处理 > 5.6 短剧视频换脸 > 5. 项目实战

## 核心概念

- **是什么**：脸部预处理是换脸前的关键步骤——在送入换脸模型之前，对人脸进行检测、对齐、以及特定处理（如闭嘴、光头化），消除干扰因素，提高换脸成功率。
- **为什么重要**：原始视频中的人脸存在角度、遮挡、表情、发型等各种干扰。不做好预处理，换脸结果会出现五官错位、边缘模糊、头发穿模等问题。

## 原理讲解

### 三种预处理方式

| 预处理 | 目的 | 方法 |
|--------|------|------|
| **人脸检测** | 找出人脸位置和关键点 | MTCNN/RetinaFace 检测 68/106 个关键点 |
| **闭嘴处理** | 消除张嘴导致的牙齿穿帮 | 检测嘴部开合度 → 选择闭嘴帧/图像修复 |
| **光头化** | 消除头发对脸部边缘的遮挡 | 语义分割 → 头发区域 → 图像修复填充 |

### 人脸检测流程

```
原始帧 → 人脸检测器 → 人脸框 + 关键点 → 仿射变换对齐 → 标准人脸(256×256)
```

为什么需要对齐？同一个人在不同帧中角度不同，对齐后统一到标准姿态，换脸模型才能稳定工作。


### 3. 脸部预处理管线

```
原帧 BGR → 检测 bbox → 对齐(5点 landmark) → 256×256 对齐脸
     → 颜色校正 → 送入换脸模型
```

**失败处理**：检测失败帧 **沿用上一帧 bbox** 或插值，避免成片空洞。

### 4. 与 5-3 AI 质检的关联

同一套检测模型可用于「人脸是否完整」质检 gate（见 5-3）。

## 代码实战

```python
# pip install numpy

"""脸部预处理流水线"""
import cv2, numpy as np

class FacePreprocessor:
    def __init__(self, detector_model="retinaface"):
        # 伪代码：实际使用 MTCNN 或 RetinaFace
        self.detector = self._load_detector(detector_model)
    
    def detect_and_align(self, frame: np.ndarray) -> list[dict]:
        """检测人脸并对齐"""
        faces = self.detector.detect(frame)
        results = []
        for face in faces:
            # 提取关键点
            landmarks = face["landmarks"]  # 5点或68点
            
            # 仿射变换对齐到标准位置
            aligned = self._align_face(frame, landmarks, target_size=(256, 256))
            
            results.append({
                "bbox": face["bbox"],           # (x, y, w, h)
                "landmarks": landmarks,
                "aligned_face": aligned,
                "confidence": face["confidence"]
            })
        return results
    
    def _align_face(self, img, landmarks, target_size=(256, 256)):
        """仿射变换——把人脸对齐到标准位置"""
        # 标准位置（基于5个关键点的目标坐标）
        std_points = np.array([
            [70, 112], [182, 112],  # 左右眼角
            [126, 168],              # 鼻尖
            [70, 200], [182, 200]   # 左右嘴角
        ])
        
        src = np.array(landmarks[:5], dtype=np.float32)
        dst = std_points.astype(np.float32)
        
        matrix = cv2.estimateAffinePartial2D(src, dst)[0]
        return cv2.warpAffine(img, matrix, target_size)
    
    def detect_mouth_open(self, landmarks) -> bool:
        """检测嘴巴是否张开——上下唇关键点距离"""
        if len(landmarks) >= 68:
            upper_lip = landmarks[62]  # 上唇中点
            lower_lip = landmarks[66]  # 下唇中点
            distance = np.linalg.norm(np.array(upper_lip) - np.array(lower_lip))
            return distance > 10  # 阈值可调
        return False
    
    def remove_hair(self, face_region: np.ndarray) -> np.ndarray:
        """去除头发——用语义分割+图像修复"""
        # 简化版本：用头发颜色阈值做粗分割
        hsv = cv2.cvtColor(face_region, cv2.COLOR_BGR2HSV)
        # 头发通常是深色
        hair_mask = cv2.inRange(hsv, (0, 0, 0), (180, 255, 80))
        
        # 用周围肤色填充头发区域（简化版 inpainting）
        result = cv2.inpaint(face_region, hair_mask, inpaintRadius=3, 
                            flags=cv2.INPAINT_TELEA)
        return result
    
    def preprocess(self, frame: np.ndarray) -> list[dict]:
        """完整预处理流水线"""
        faces = self.detect_and_align(frame)
        
        for face in faces:
            face["mouth_open"] = self.detect_mouth_open(face["landmarks"])
            face["hair_removed"] = self.remove_hair(face["aligned_face"])
        
        return faces
```

## 进阶方向

### 当前方案的局限性

- **5点对齐精度不足**：5 点关键点（眼角+鼻尖+嘴角）只能做粗略仿射变换，大角度侧脸需要 68 点做更精细的透视变换
- **颜色阈值粗分割**：用 HSV 阈值区分头发，不同人的头发颜色差异大（黑色/棕色/金色），固定阈值不通用
- **图像修复质量**：cv2.inpaint 的 TELEA 算法在头发大面积遮挡时效果差，需要深度学习 inpainting（如 LaMa）
- **实时性差**：检测+对齐+预处理每帧耗时 50-100ms，24fps 视频无法实时处理

### 工业界最佳实践

- **关键点密度选择**：正面脸用 5 点（快速），侧脸用 68 点（高精度），极端角度用 106 点
- **预处理缓存**：同一视频的预处理结果（人脸框、关键点、对齐脸）缓存到磁盘，重跑时直接加载
- **处理失败降级**：预处理失败（如检测不到人脸）时，该帧原样保留而非丢弃，避免视频中出现黑帧

## 常见问题

### 小白最常踩的 3 个坑

**坑 1：对齐后人脸变形**

仿射变换参数算错——源关键点和目标点不匹配。5 点源顺序必须是"左眼、右眼、鼻尖、左嘴角、右嘴角"，和目标点一一对应。顺序错了，对齐结果就是扭曲的。

**坑 2：闭嘴检测逻辑粗糙**

只用上下唇距离判断张嘴，忽略了"微张嘴"（说话时自然状态）。应该用唇部面积比——张开面积/闭合面积 > 1.5 才算张嘴。

**坑 3：光头化把眉毛也抹掉了**

头发分割的 HSV 阈值和深色眉毛接近，inpaint 时眉毛也一起被填充了，换脸结果看起来像"无眉大侠"。应该先检测眉毛区域（通常在人脸上半部），在头发 mask 中排除眉毛区域。

### 自检题

**Q1**：三种脸部预处理方式分别解决什么问题？<details><summary>答案</summary>人脸检测+对齐→解决角度/位置不一致。闭嘴处理→消除张嘴牙齿穿帮。光头化→消除头发遮挡脸部边缘。</details>

**Q2**：为什么人脸对齐如此重要？<details><summary>答案</summary>同一人在不同帧中角度各异，换脸模型期望标准姿态的输入。通过仿射变换将所有脸对齐到统一的256×256标准位置，模型才能稳定输出一致的结果。</details>

**Q3**：什么时候用 5 点对齐，什么时候用 68 点？<details><summary>答案</summary>正面脸（偏转角 < 30°）用 5 点快速对齐；侧脸（偏转角 30°-60°）用 68 点做更精细变换；极端角度（> 60°）用 106 点或直接跳过该帧。</details>
