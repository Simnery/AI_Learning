# 5.3 AI质检

> 5.3 AI质检 > 5. 项目实战

## 核心概念

- **是什么**：AI 质检是用计算机视觉技术自动检测工业产品表面缺陷。本项目对比 YOLO（专用检测模型）和 Qwen-VL（多模态大模型）在工业质检中的表现，揭示各自的适用边界。
- **为什么重要**：工业质检是 AI 落地最成熟的场景之一。理解 YOLO vs 多模态大模型的优劣势对比，帮你在实际项目中做出正确的技术选型。
- **前置知识**：YOLO 基础（参考 [3-5-4 YOLO](../03_AI框架及工具平台/3-5_PyTorch与视觉检测/3-5-4_视觉检测模型YOLO.md)）、图像识别基础（参考 [3-5-3 图像识别与缺陷检测](../03_AI框架及工具平台/3-5_PyTorch与视觉检测/3-5-3_图像识别技术与缺陷检测.md)）。

## 原理讲解

### 1. 技术选型对决：YOLO vs Qwen-VL

| 维度 | YOLO | Qwen-VL |
|------|------|---------|
| 原理 | CNN 专用检测头 | Transformer 多模态理解 |
| 需要标注 | 是（每个缺陷类型需 N 张标注） | 零样本或少量标注 |
| 速度 | 极快（实时） | 较慢（秒级） |
| 精度（有足够标注） | 高（mAP > 0.9） | 中（取决于 prompt） |
| 复杂缺陷描述 | 只能分类+定位 | 能描述缺陷特征、原因推断 |
| 部署 | 轻量（ONNX/TensorRT） | 重量（需要 GPU） |
| 成本 | 标注成本高 | 推理成本高 |

### 2. 选型决策树

```
质检需求是什么？
├── 缺陷类型固定 + 有大量标注 → YOLO（精度最高，速度最快）
├── 缺陷类型多变 + 标注困难 → Qwen-VL（零样本，灵活）
└── 需要缺陷描述/原因分析 → Qwen-VL（多模态理解）
```

### 3. 数据工程——质检项目的成败关键

```
原始数据 → EDA分析 → 数据清洗 → 格式转换 → 划分训练/验证/测试
                                                      ↓
                                            YOLO格式: class_id x y w h(归一化)
```

EDA 要关注：缺陷类别分布、图像尺寸分布、标注质量（是否有漏标/错标）、类别不平衡程度。

## 代码实战

### 示例1：数据集 EDA

```python
# pip install — 本示例仅用 Python 3.10+ 标准库

"""质检数据集探索性分析"""
import os, json
from collections import Counter

def eda_analysis(annotations_dir: str):
    """分析缺陷数据集的统计特征"""
    stats = {"total_images": 0, "total_defects": 0, "class_dist": Counter(),
             "img_sizes": [], "defects_per_img": []}
    
    for ann_file in os.listdir(annotations_dir):
        with open(os.path.join(annotations_dir, ann_file)) as f:
            data = json.load(f)
        
        stats["total_images"] += 1
        n_defects = len(data.get("objects", []))
        stats["total_defects"] += n_defects
        stats["defects_per_img"].append(n_defects)
        
        for obj in data.get("objects", []):
            stats["class_dist"][obj["label"]] += 1
    
    print(f"图片总数: {stats['total_images']}")
    print(f"缺陷总数: {stats['total_defects']}")
    print(f"平均每图缺陷数: {np.mean(stats['defects_per_img']):.1f}")
    print(f"类别分布: {stats['class_dist'].most_common()}")
    return stats
```

### 示例2：Qwen-VL 零样本缺陷检测

```python
"""用 Qwen-VL 做零样本缺陷检测——无需标注"""
from transformers import Qwen2VLForConditionalGeneration, AutoProcessor

model = Qwen2VLForConditionalGeneration.from_pretrained(
    "Qwen/Qwen2-VL-7B-Instruct", torch_dtype="auto", device_map="auto"
)
processor = AutoProcessor.from_pretrained("Qwen/Qwen2-VL-7B-Instruct")

def zero_shot_defect_detection(image_path: str) -> str:
    """零样本缺陷检测——不需要训练，直接问大模型"""
    from PIL import Image
    image = Image.open(image_path)
    
    messages = [{
        "role": "user",
        "content": [
            {"type": "image", "image": image},
            {"type": "text", "text": """这是一张工业产品表面图。
请检测是否有以下缺陷：划痕、裂纹、污点、凹陷。
如果有缺陷，请描述类型、位置和严重程度。
如果无缺陷，回答"良品"。

输出格式: JSON {"defects": [{"type":"...", "location":"...", "severity":"...", "bbox": [x1,y1,x2,y2]}]}"""}
        ]
    }]
    
    inputs = processor.apply_chat_template(messages, return_tensors="pt").to(model.device)
    outputs = model.generate(inputs, max_new_tokens=500)
    return processor.decode(outputs[0], skip_special_tokens=True)
```

## 进阶方向

### 当前方案的局限性

| 局限 | 说明 |
|------|------|
| YOLO 泛化能力 | YOLO 模型只认识训练过的缺陷类型，出现新的缺陷形态时可能漏检 |
| Qwen-VL 幻觉 | Qwen-VL 可能"看到"不存在的缺陷，或者把正常的纹理误判为裂纹 |
| 小样本缺陷 | 罕见缺陷（如异色斑点）只有 3-5 个样本，CNN 很难学到有效特征 |
| 边缘部署 | 工厂产线上只有嵌入式设备，Qwen-VL 根本跑不动，必须用小模型 |

### 工业界最佳实践

- **分级检测策略**：第一级用 YOLOv8-nano 做快速初筛（识别明显缺陷），第二级用 Qwen-VL 对模糊样本做二次判断，兼顾速度和精度
- **缺陷知识库**：把每次检测出的缺陷图像、类型、处理结果存入知识库，形成"缺陷百科"，供后续分析和模型迭代
- **人工复核闭环**：低置信度（0.5-0.7）的检测结果推送给质检员确认，确认结果自动加入训练集

## 常见问题

### 小白最常踩的 3 个坑

**坑 1：直接拿公开数据集训练，不检查本厂数据的分布差异**

公开数据集（如 NEU-DET 钢铁缺陷）和你工厂的实际缺陷图可能完全不同——光照、角度、缺陷形态都有差异。直接用公开模型不做适配，准确率可能不到 50%。必须收集自己工厂的数据做微调。

**坑 2：只关注检出率，不看误报率**

模型把所有产品都标为"有缺陷"能达到 100% 检出率——但误报率 90%，产线根本没法用。质检模型真正的挑战是在检出率和误报率之间找平衡。一般要求检出率 > 95% 且误报率 < 5%。

**坑 3：忽略光照一致性问题**

工业相机在不同时间（上午 vs 下午）拍同一产品出来的图像亮度可能完全不同。CV 模型对光照敏感——训练时光照充足，推理时光照暗，模型就"瞎了"。必须做光照归一化预处理或数据增强时加入光照变化。

### 自检题

**Q1**：什么场景选 YOLO，什么场景选 Qwen-VL？<details><summary>答案</summary>YOLO：缺陷类型固定、有足够标注、需要实时检测（毫秒级）、边缘部署。Qwen-VL：缺陷类型多变、标注困难/成本高、需要缺陷描述和原因推断、计算资源充足。</details>

**Q2**：质检数据工程中最关键的三个步骤是什么？<details><summary>答案</summary>1) EDA 分析（了解类别分布、标注质量、数据特点）2) 格式转换（统一标注格式→YOLO/COCO）3) 类别平衡处理（过采样/数据增强解决罕见缺陷样本不足）。</details>

**Q3**：Qwen-VL 的"零样本"检测有什么优势和局限？<details><summary>答案</summary>优势：不需要标注，上线快，能识别训练集没有的新缺陷类型。局限：精度不如专用训练模型、速度慢、对 prompt 敏感、可能产生幻觉。</details>
