## 核心概念

### 什么是多模态 RAG

**一句话**：RAG 不仅能处理纯文本，还能"看懂"PDF 排版、"读懂"图片图表、"听懂"音频内容——把各种格式的数据都变成 LLM 可理解的文本/向量。

```
纯文本 RAG：txt → Embedding → 检索 → 生成
多模态 RAG：PDF/Word/网页/图片/视频/音频 → 解析提取 → Embedding → 检索 → 生成
```

**类比**：纯文本 RAG 是只能读 .txt 的记事本；多模态 RAG 是能打开任何格式的"万能阅读器"。

### 为什么多模态数据处理重要

现实世界的知识只有一小部分是纯文本。企业知识库充斥着：

| 格式 | 典型场景 | 挑战 |
|------|---------|------|
| **PDF** | 论文、报告、手册 | 双栏排版、表格、扫描件 OCR |
| **Word** | 需求文档、会议纪要 | 样式嵌套、修订痕迹、嵌入图片 |
| **网页** | 技术博客、官方文档 | JS 动态渲染、导航栏噪音、反爬 |
| **图片** | 架构图、数据图表、截图 | 文字提取(OCR)、视觉理解 |
| **视频** | 教程、会议录制 | 帧采样、语音转文字、场景切换 |
| **音频** | 播客、语音消息 | ASR 准确度、多说话人分离 |

## 原理讲解

### PDF 解析：看起来简单，实际很复杂

PDF 是"打印格式"，不是"内容格式"——它记录的是"这个字符在页面的 (x, y) 位置"，而不是"这是第 3 段的第 2 句话"。

**PDF 解析的三层挑战**：

```
第1层 - 文本提取
  "这个字符在 (120, 350)" → 还原成句子
  问题: 双栏排版会读成"第1栏第1行 第2栏第1行 第1栏第2行..."
  
第2层 - 结构识别
  哪些是标题？哪些是正文？哪些是表格？
  问题: PDF 没有"标题"概念，只有"字体大一点的文字"
  
第3层 - 语义理解
  表格里的数据是什么意思？
  问题: 跨页表格、合并单元格、扫描件 OCR 识别
```

**主流解析策略**：

| 策略 | 工具 | 适用场景 |
|------|------|---------|
| 规则解析 | PyMuPDF, pdfplumber | 电子原生 PDF，排版规整 |
| OCR 解析 | Tesseract, PaddleOCR | 扫描件、图片型 PDF |
| 模型解析 | Docling, Marker, MinerU | 复杂排版（论文、双栏） |
| 云服务 | 阿里云/百度云文档解析 | 高精度需求，不想折腾 |

**推荐方案**：先用 PyMuPDF 快速提取，遇到乱序/表格再上模型解析（Docling/MinerU）。

### Word 文档解析

Word (.docx) 本质是 XML 压缩包，比 PDF 友好得多：

```python
# .docx 文件其实就是这个结构：
word/
  document.xml    ← 正文内容
  styles.xml      ← 样式定义
  media/          ← 嵌入的图片
```

解析要点：
- **样式 → 结构**：标题样式(Heading 1/2/3)直接映射到 Markdown 标题
- **表格**：python-docx 可直接读取行列，需还原为 Markdown 表格
- **嵌入图片**：提取后用多模态模型（GPT-4o/Claude）描述图片内容

### 网页数据解析

网页比 PDF/Word 更"脏"——导航栏、广告、推荐列表都是噪音。

**清洗流程**：
```
原始 HTML → 去除 script/style/nav/footer → 提取正文(content) → Markdown 化
```

常用工具链：
- **html2text / trafilatura**：自动提取正文，去除噪音
- **BeautifulSoup + 自定义规则**：对特定网站精细提取
- **Jina AI Reader API**：一键获取任何网页的干净 Markdown

### 图像数据的多重表征

图片不能直接 Embedding（文本 Embedding 模型不认图像），需要"翻译"成文本：

**三种表征方式**：

| 方式 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **OCR 提取文字** | 检测图片中的文字 → 转成文本 | 速度快，成本低 | 丢失视觉布局信息 |
| **Caption 描述** | 多模态模型看图说话 | 保留视觉语义 | 成本较高 |
| **多模态 Embedding** | CLIP 等模型直接对图片向量化 | 图文联合检索 | 中文支持有限 |

**最佳实践**：OCR + Caption 双路并存——
```
图片 → OCR 提取文字 → 存入 text 字段
图片 → 多模态模型描述 → 存入 caption 字段
检索时：text(权重0.3) + caption(权重0.7) 混合匹配
```

### 视频数据处理流程

视频 = 图像序列 + 音频流 + 可能的字幕，处理需**多管线并行**：

```
视频文件
  ├─→ 关键帧采样(每2秒一帧) → 图像Caption → 场景描述文本
  ├─→ 音频提取 → ASR → 对话文本
  └─→ 字幕提取(如有) → 时间戳对齐文本
        ↓
  三者按时间戳合并 → 结构化文档 → Embedding → 向量库
```

### 语音转文本（ASR）

| 方案 | 适用场景 | 速度 | 准确度 |
|------|---------|:----:|:------:|
| **Whisper** (OpenAI) | 通用，多语言 | 中等 | 高 |
| **FunASR** (阿里) | 中文优化，支持热词 | 快 | 中文极高 |
| **SenseVoice** (阿里) | 多语言 + 情感识别 | 快 | 高 |
| **Paraformer** | 非自回归，流式 | 极快 | 中等 |

**RAG 场景推荐**：FunASR（中文场景）或 Whisper Large v3（多语言），输出带时间戳，便于与视频帧对齐。

### GraphRAG：用知识图谱增强 RAG

传统 RAG 做不了的事：**多跳推理和全局理解**。

```
问题: "公司2024年所有AI项目中，哪个ROI最高？"

传统 RAG:
  检索到"Q1项目 ROI 15%""Q3项目 ROI 22%"等碎片
  → 无法综合对比

GraphRAG:
  实体关系图: 项目A─→ROI─→15%, 项目C─→ROI─→22%
  → 遍历图，找到 ROI 最高的节点
```

**两种搜索模式**：

| 模式 | 做什么 | 适用问题 |
|------|--------|---------|
| **局部搜索** | 从某个实体出发，探索其邻居节点 | "项目A用到了哪些技术？" |
| **全局搜索** | 在整个图上找跨社区的模式和主题 | "公司整体技术栈有哪些趋势？" |

**GraphRAG 使用要点**：
1. 文档 → LLM 提取实体和关系 → 构建知识图谱
2. 社区检测（Leiden 算法）→ 为每个社区生成摘要
3. 查询时结合局部搜索（精确）和全局搜索（宏观）

### 实战 1：PDF 解析 + 表格提取

```python
"""PDF 解析：提取文本、表格，按段落切分"""
import fitz  # PyMuPDF

def parse_pdf(pdf_path):
    """解析 PDF，返回页面列表，每个页面含文本和表格"""
    doc = fitz.open(pdf_path)
    pages = []

    for page_num in range(len(doc)):
        page = doc[page_num]
        blocks = page.get_text("blocks")  # 按块提取，保留位置信息

        # 按 y 坐标排序（从上到下），按 x 坐标判断是否双栏
        blocks_sorted = sorted(blocks, key=lambda b: (b[1], b[0]))

        page_text = []
        for b in blocks_sorted:
            if b[6] == 0:  # 文本块
                text = b[4].strip()
                if len(text) > 3:  # 过滤太短的（页码等）
                    page_text.append(text)

        # 提取表格
        tables = page.find_tables()
        table_data = []
        for table in tables:
            table_data.append(table.extract())

        pages.append({
            "page": page_num + 1,
            "text": "\n".join(page_text),
            "tables": table_data,
        })
        print(f"第{page_num+1}页: {len(page_text)} 文本块, {len(table_data)} 表格")

    doc.close()
    return pages

# 使用示例

# 无 PDF 时的演示
```

### 实战 2：网页内容抓取 + 清洗

```python
"""从网页提取正文，去除导航/广告等噪音"""
import trafilatura

def fetch_webpage(url):
    """抓取网页并提取干净的正文 Markdown"""
    # trafilatura 自动处理下载+解析+清洗
    downloaded = trafilatura.fetch_url(url)
    if downloaded is None:
        print(f"抓取失败: {url}")
        return None

    # 提取正文（含格式），输出 Markdown
    text = trafilatura.extract(
        downloaded,
        output_format="markdown",
        include_links=True,
        include_images=False,    # 图片 URL 噪音多，建议关闭
        include_tables=True,
    )

    # 提取元数据
    metadata = trafilatura.extract(
        downloaded,
        output_format="json",
        include_formatting=False,
    )

    return {"content": text, "url": url, "title": metadata.get("title", "")}

# 测试
```

### 当前技术的局限

1. **PDF 表格解析不稳定**：跨页表格、合并单元格、无边框表格仍是难点
2. **视频理解成本高**：1 小时视频用 GPT-4o 逐帧分析成本约 $5-10

## 常见问题

### 小白最常踩的 3 个坑

1. **PDF 解析直接用 PyPDF2 读，结果乱序**
   - 错误：PyPDF2 逐字符按位置排序，遇到双栏/表格输出完全乱序
   - 正确：用 PyMuPDF（fitz）的 `get_text("blocks")` 按阅读顺序提取，或直接用 Docling/MinerU 等模型方案

2. **网页抓取没去噪音，全文塞进知识库**
   - 错误：把导航栏"首页 关于 联系我们"、广告横幅、推荐列表都 Embedding 了
   - 正确：用 trafilatura 或 Jina Reader 提取正文；或者用 BeautifulSoup 手动去除 `<nav>` `<footer>` `<script>` 等标签

3. **图片只做 OCR，忽略了视觉含义**
   - 错误：一张"系统架构图"，OCR 只提取了"输入 输出 模型"，丢失了组件间的连接关系
   - 正确：OCR 文字 + 多模态模型描述 双路并存。描述包含"这是一张系统架构图，展示了从输入到输出的三层流水线..."

### 自检题

**Q1**：为什么 PDF 解析比 Word 解析难得多？根本原因是什么？

> **答案**：PDF 是"打印格式"——它记录的是字符在页面上的 (x, y) 坐标和字体大小，没有"段落""标题""表格"这些结构概念。Word (.docx) 本质是 XML，天然有样式层级。PDF 解析要做"反向工程"从视觉布局还原语义结构。

**Q2**：处理一张"数据图表"时，OCR 和 Caption 分别能提供什么信息？为什么要双路？

> **答案**：OCR 提取图表中的具体数字和标签（"2024 年增长率 35.2%"）；Caption 理解图表的类型和趋势（"这是一张柱状图，展示了 2020-2024 年的增长趋势，整体呈上升趋势"）。双路互补：用户搜"增长率趋势"时 Caption 能命中，搜"35.2%"时 OCR 能命中。

**Q3**：GraphRAG 的"局部搜索"和"全局搜索"有什么区别？什么问题时用全局搜索？

> **答案**：局部搜索从具体实体出发探索其邻居（"项目A用了什么技术栈？"）；全局搜索在整个图上跨社区找更高层次的主题模式（"公司技术投入的整体趋势是什么？"）。需要宏观总结、趋势分析、对比总结时用全局搜索。
