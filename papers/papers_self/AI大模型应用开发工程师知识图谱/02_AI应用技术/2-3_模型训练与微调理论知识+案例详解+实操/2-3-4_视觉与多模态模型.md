## 核心概念

### 什么是多模态模型

**一句话**：多模态模型能同时理解和处理文字、图片、音频、视频等多种类型的数据，不仅是"看图说话"，而是真正"理解"视觉世界。

```
单模态 AI:  文字 → LLM → 文字
多模态 AI:  文字+图片+音频+视频 → VLM → 综合分析输出
```

**类比**：单模态 AI 是盲人学者——知识渊博但看不见世界。多模态 AI 是正常的专家——既能读书又能看图、听声音，全面理解信息。

### 视觉识别的三大核心任务

| 任务 | 输入 → 输出 | 典型应用 |
|------|-----------|---------|
| **图像分类** | 图片 → "猫"/"狗" | 照片自动分类、内容审核 |
| **目标检测** | 图片 → "猫在(x1,y1,x2,y2)" | 自动驾驶、安防监控、缺陷检测 |
| **图像分割** | 图片 → 每个像素的类别 | 医学影像、自动驾驶路面分割 |

```
图像分类:  整张图 → 一个标签
目标检测:  图中多个物体 → 每个物体的位置+类别(框出来)
图像分割:  图中每个像素 → 属于什么(精确到像素级)
```

## 原理讲解

### VLM（视觉语言模型）的工作原理

现代 VLM 的核心架构：**视觉编码器 + 连接器 + LLM**

```
图片 → 视觉编码器(如 CLIP ViT) → 图像 Token
                                        ↓ 连接器(MLP/Cross-Attention)
                                        ↓ 映射到 LLM 的 Embedding 空间
文本 → Tokenizer → 文本 Token → [文本Token + 图像Token] → LLM → 回答
```

**主流 VLM 对比**：

| 模型 | 架构特点 | 视觉编码器 | 适用场景 |
|------|---------|-----------|---------|
| **GPT-4o** | 原生多模态 | 未公开 | 全能，端到端多模态理解 |
| **Qwen-VL** | ViT + Qwen LLM | ViT-G | 中文多模态 |
| **LLaVA** | CLIP ViT + Llama | CLIP ViT-L | 开源标杆，学术研究 |
| **InternVL2** | 动态分辨率 ViT | InternViT | 高分辨率文档理解 |

### 视频理解的核心挑战

视频比图片多了"时间"维度：

```
图片理解: 空间信息(画面中有什么)
视频理解: 空间信息 + 时间信息(发生了什么, 顺序是什么)

处理方式:
  1. 关键帧采样: 每 N 秒取一帧 → 多帧图像理解
  2. 3D 卷积: 同时在空间和时间维度卷积
  3. 时序 Transformer: 帧序列作为输入 → 学习时间依赖
```

### YOLO 的核心原理

YOLO（You Only Look Once）如其名——**一次前向传播就完成检测**：

```
传统方法(两阶段):
  1. 找候选区域(Region Proposal) → 2. 对每个区域分类
  慢但准, 如 Faster R-CNN

YOLO(单阶段):
  整张图 → 卷积网络 → 同时输出: 边界框 + 类别 + 置信度
  快但需要更多数据
```

**YOLO 的工作流程**：

```
1. 将图片划分为 S×S 的网格
2. 每个网格预测 B 个边界框(位置+大小)和 C 个类别概率
3. 过滤低置信度的框 → 非极大值抑制(NMS)去重
4. 输出最终检测结果
```

### 智能文档模型 MinerU

MinerU 是专门针对 PDF/文档的视觉理解模型：

```
传统 PDF 解析: 规则提取文本(遇到双栏→乱序, 遇到表格→丢失)
MinerU:         视觉理解 PDF(像人一样"看"页面布局)

核心技术:
  - 版面分析: 识别标题/正文/表格/图片/公式区域
  - 公式识别: LaTeX 公式 OCR(数学论文的关键)
  - 表格识别: 将表格还原为 Markdown/HTML
  - 阅读顺序恢复: 理解人类阅读顺序(标题→第1栏→第2栏)
```

### 实战 2：训练 YOLO 目标检测模型

```python
"""YOLO 目标检测模型训练"""
print("""
# === YOLOv8 训练自定义检测模型 ===
from ultralytics import YOLO

# ===== 1. 准备数据集 =====
# 数据集目录结构:
# dataset/
#   images/
#     train/  ← 训练图片
#     val/    ← 验证图片
#   labels/
#     train/  ← 训练标注(每张图一个.txt)
#     val/    ← 验证标注

# 标注格式(每行一个目标):
# class_id x_center y_center width height  (值都是归一化到 0-1)
# 例如: 0 0.523 0.456 0.123 0.234

# ===== 2. 创建数据配置 =====
import yaml

data_config = {
    "path": "./dataset",
    "train": "images/train",
    "val": "images/val",
    "names": {0: "缺陷", 1: "划痕", 2: "污点"},  # 你的类别
}

with open("data.yaml", "w") as f:
    yaml.dump(data_config, f)

# ===== 3. 训练 =====
# 加载预训练模型
model = YOLO("yolov8n.pt")  # n=nano(最小), s=small, m=medium, l=large, x=xlarge

# 训练
results = model.train(
    data="data.yaml",
    epochs=100,
    imgsz=640,             # 图片尺寸
    batch=16,
    lr0=0.01,             # 初始学习率
    device="cuda",        # 或 "cpu"
    project="./yolo_output",
    name="defect_detector",
)

# ===== 4. 评估 =====
metrics = model.val()
print(f"mAP50: {metrics.box.map50:.3f}")       # 平均精度(IoU=0.5)
print(f"mAP50-95: {metrics.box.map:.3f}")       # 平均精度(IoU=0.5-0.95)
print(f"Precision: {metrics.box.mp:.3f}")       # 精确率
print(f"Recall: {metrics.box.mr:.3f}")          # 召回率

# ===== 5. 推理 =====
results = model.predict(
    source="test_image.jpg",
    conf=0.25,            # 置信度阈值
    save=True,            # 保存标注结果
)
# results[0].show()       # 展示结果
''')

print("YOLO 训练关键参数说明:")
print("  - mAP50: 当预测框与真实框 IoU>0.5 时的平均精度(最常用指标)")
print("  - mAP50-95: 多个 IoU 阈值的综合指标(更严格)")
print("  - 模型大小: nano(最轻) → small → medium → large → xlarge(最重)")
print("  - 建议: 先 nano 快速验证, 效果好再上 large")
```

### 实战 3：目标检测数据集标注

```python
"""了解目标检测标注格式"""
import json

print("""
# === 目标检测标注工具 ===
# 推荐工具:
#   - LabelImg: 经典, 界面简单
#   - Label Studio: 功能丰富, 支持团队协作
#   - CVAT: Intel 出品, 视频标注强
#   - Roboflow: 在线平台, 自动标注辅助

# === 标注格式示例 ===

# YOLO 格式(label.txt, 每行一个目标):
# class_id center_x center_y width height
0 0.523 0.456 0.123 0.234
1 0.710 0.320 0.089 0.156

# COCO 格式(JSON):
{
  "images": [{"id": 1, "file_name": "img001.jpg", "width": 640, "height": 480}],
  "annotations": [
    {
      "id": 1,
      "image_id": 1,
      "category_id": 0,
      "bbox": [250, 180, 80, 60],  // [x, y, width, height] 绝对坐标
      "area": 4800
    }
  ],
  "categories": [
    {"id": 0, "name": "缺陷"},
    {"id": 1, "name": "划痕"}
  ]
}
''')

# 标注格式转换工具
def yolo_to_coco(yolo_line, img_w, img_h):
    """YOLO 格式 → COCO 格式"""
    parts = yolo_line.strip().split()
    cls_id = int(parts[0])
    cx, cy, w, h = map(float, parts[1:5])
    # 从中心点+宽高 → 左上角+宽高
    x = (cx - w/2) * img_w
    y = (cy - h/2) * img_h
    w = w * img_w
    h = h * img_h
    return {"category_id": cls_id, "bbox": [round(x), round(y), round(w), round(h)]}

# 演示
demo = "0 0.523 0.456 0.123 0.234"
result = yolo_to_coco(demo, 640, 480)
print(f"YOLO: {demo}")
print(f"COCO: {result}")
```

## 进阶方向

### 多模态模型的趋势

1. **原生多模态**：GPT-4o 不再是"视觉+LLM的拼接"，而是用同一个模型同时处理所有模态
2. **任意分辨率**：支持高分辨率输入而不需要 resize（InternVL2 的动态分辨率方案）
3. **视频理解升级**：从"逐帧看图"到"理解动作和事件序列"
4. **文档专用模型**：MinerU/Docling 等针对 PDF/扫描件的垂直优化

### YOLO 训练优化技巧

| 技巧 | 说明 |
|------|------|
| **数据增强** | 随机翻转、旋转、HSV 色彩抖动，防止过拟合 |
| **预训练权重** | 先加载 COCO 预训练，再在自己的数据上微调 |
| **小目标检测** | 提高输入分辨率(640→1280)，或专门训练小目标检测器 |
| **类别不平衡** | 过采样少样本类别 + 类别加权损失函数 |
| **困难样本挖掘** | 收集模型容易出错的样本，针对性补充训练数据 |

## 常见问题

### 小白最常踩的 3 个坑

1. **标注格式搞混**
   - 错误：YOLO 需要归一化坐标(0-1)，却给了绝对坐标(像素值)
   - 正确：检查标注值范围——YOLO 格式所有坐标必须是 0-1 之间的浮点数

2. **训练图片尺寸不一致直接训练**
   - 错误：数据集里有 200×300 和 4000×6000 的图片混在一起
   - 正确：YOLO 会自动 resize，但长宽比差异太大时应先预处理(crop/pad)

3. **VLM 理解图片时期望太高**
   - 错误：期望 VLM 能精确识别"图片右下角的6号宋体小字"
   - 正确：VLM 对高分辨率细节和微小文字的识别力有限，关键信息尽量在图中放大

### 自检题

**Q1**：YOLO "一次前向传播"完成为什么是一个优势？

> **答案**：两阶段检测器（如 Faster R-CNN）需要先找候选区域再分类，两步骤都需要前向传播，速度慢。YOLO 将"找目标+分类"合并为一个端到端的网络，一次推理就完成所有操作，是实时检测（30+ FPS）的关键。代价是需要更多训练数据和更细心的调参来达到两阶段检测器的精度。

**Q2**：为什么 PDF 解析需要专门的视觉模型（如 MinerU）而不能只用 OCR？

> **答案**：OCR 只提取文字位置，不理解"文档结构"。比如一个双栏 PDF，OCR 可能按从左到右的顺序提取：第一栏的"第1章 引言" + 第二栏的"...技术飞速发展" → 读成一句，完全不可读。MinerU 理解"人类是怎么阅读的"——它识别出双栏布局，先读左栏再读右栏，还识别出公式（LaTeX）、表格（Markdown）、标题层级等结构化信息。

**Q3**：多模态模型中的"视觉编码器+连接器+LLM"架构，连接器的作用是什么？

> **答案**：连接器负责"翻译"——视觉编码器输出的是图像特征（高维向量），LLM 理解的是文本 token。这两者处于完全不同的"语义空间"。连接器（通常是一个简单的 MLP 或 Cross-Attention 层）将图像特征投影到文本 token 的空间，让 LLM 能像理解文字一样理解图像。就像给两个说不同语言的人之间配一个翻译。
