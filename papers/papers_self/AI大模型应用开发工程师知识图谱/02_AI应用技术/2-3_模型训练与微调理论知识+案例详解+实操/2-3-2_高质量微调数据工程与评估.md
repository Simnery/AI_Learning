## 核心概念

### 什么是微调数据工程

**一句话**：微调效果 80% 取决于数据质量，而非训练参数。数据工程是微调中最重要、最耗时的环节。

```
微调成功的公式:
  好数据 + 差训练配置 > 差数据 + 好训练配置

Garbage In, Garbage Out:
  用低质量数据微调 → 模型学会"坏习惯" → 效果反而比基座模型更差
```

**类比**：微调是"师傅带徒弟"——数据就是师傅教的内容。师傅教得乱七八糟，徒弟再聪明也学不好。

### SFT vs RLHF 数据的本质区别

| 维度 | SFT 数据 | RLHF/DPO 数据 |
|------|---------|--------------|
| **是什么** | 标准答案（"这个问题应该这样答"） | 偏好比较（"这个回答比那个好"） |
| **格式** | `{instruction, output}` | `{instruction, chosen, rejected}` |
| **目标** | 教会模型"怎么做" | 教会模型"什么更好" |
| **来源** | 人工撰写/LLM 生成+人工审核 | 人工排序/用户反馈 |
| **数据量** | 1000-50000 条 | 500-10000 对 |

## 原理讲解

### 数据收集策略

**四种数据来源对比**：

| 来源 | 质量 | 成本 | 适用场景 |
|------|:----:|:----:|---------|
| **人工撰写** | 最高 | 最高（¥50-200/条） | 专业领域、高精度需求 |
| **LLM 生成 + 人工审核** | 高 | 中（¥1-5/条） | 通用场景，最推荐 |
| **纯 LLM 生成** | 中 | 最低（¥0.01/条） | 快速原型验证 |
| **线上真实数据清洗** | 不定 | 低 | 有生产环境的场景 |

**推荐组合**：LLM 生成 + 人工审核（性价比最优）。LLM 负责"量大"，人负责"把关"。

### 数据清洗核心流程

```
原始数据
  ↓
1. 去重 ─ 完全相同或高度相似的样本 → 只保留 1 条
  ↓
2. 格式检查 ─ 无效 JSON、缺失字段、乱码 → 删除或修复
  ↓
3. 内容过滤 ─ 包含敏感词、个人隐私、过长/过短 → 过滤
  ↓
4. 质量评分 ─ LLM 自动打分(1-5分) → 过滤 < 3 分的
  ↓
5. 多样性检查 ─ 按主题/任务类型聚类 → 补充缺失类别
  ↓
6. 人工抽检 ─ 随机 5-10% 人工复核 → 计算合格率
  ↓
清洗后的高质量数据集
```

### 数据标注规范

**好的微调数据长什么样**：

```
✅ 高质量示例:
{
  "instruction": "将以下会议纪要提炼为 3 条关键决策",
  "input": "2024Q3 会议讨论了...(300 字会议记录)",
  "output": "1. 决定将 AI 项目预算提高至 500 万\n2. ...\n3. ..."
}
要点: 指令明确、输入有上下文、输出格式清晰

❌ 低质量示例:
{
  "instruction": "帮我总结",
  "input": "...",
  "output": "好的，我来总结一下...(冗长啰嗦，无结构化)"
}
问题: 指令模糊、输出格式不一致、无收束
```

**三大标注原则**：
1. **一致性**：相同类型的问题，回答格式必须统一（不能有时用表格有时用段落）
2. **完整性**：回答要覆盖问题所有方面，不遗漏（不能问三个点只回答两个）
3. **忠实性**：回答不能编造信息（基于 input 回答，不能无中生有）

### SFT vs RLHF 数据处理流程

```
SFT 数据流程:
  原始问答 → 清洗 → 格式标准化 → 质量审核 → SFT数据集

RLHF/DPO 数据流程:
  同一问题 → 生成 2-4 个答案 → 人工排序(最好↔最差)
  → 形成 chosen/rejected 对 → DPO 数据集

DPO 的优势:
  不需要单独训练奖励模型(Reward Model)
  直接比较"好答案 vs 坏答案"
  实现更简单，效果接近 RLHF
```

### 数据质量自动评估

| 指标 | 测什么 | 计算方法 |
|------|--------|---------|
| **完整性** | 回答是否覆盖了问题要求 | LLM 逐条检查 |
| **准确性** | 回答中事实是否正确 | 与参考答案对比/LLM Judge |
| **一致性** | 同一问题多个回答风格是否一致 | 聚类分析 |
| **多样性** | 数据是否覆盖了足够多场景 | 主题分布统计 |

### 实战 1：微调数据清洗工具

```python
"""微调数据质量检查与清洗"""
import json
import hashlib
from collections import Counter

class FinetuneDataCleaner:
    """微调数据清洗器"""

    def __init__(self, min_instruction_len=10, min_output_len=20,
                 max_output_len=2000):
        self.min_instruction_len = min_instruction_len
        self.min_output_len = min_output_len
        self.max_output_len = max_output_len
        self.stats = Counter()

    def check_sample(self, sample):
        """检查单条数据质量，返回 (是否通过, 问题列表)"""
        issues = []

        # 1. 结构完整性
        if "instruction" not in sample:
            issues.append("缺少 instruction 字段")
        if "output" not in sample:
            issues.append("缺少 output 字段")

        if issues:
            self.stats["missing_fields"] += 1
            return False, issues

        inst = sample.get("instruction", "")
        out = sample.get("output", "")
        inp = sample.get("input", "")

        # 2. 长度检查
        if len(inst) < self.min_instruction_len:
            issues.append(f"instruction 太短({len(inst)} < {self.min_instruction_len})")
            self.stats["short_instruction"] += 1

        if len(out) < self.min_output_len:
            issues.append(f"output 太短({len(out)} < {self.min_output_len})")
            self.stats["short_output"] += 1

        if len(out) > self.max_output_len:
            issues.append(f"output 太长({len(out)} > {self.max_output_len})")
            self.stats["long_output"] += 1

        # 3. 内容质量(简单检查)
        if inst.strip().endswith("?") and not inst.strip().endswith("? "):
            pass  # 正常的问句

        # 拒绝模板(输出是 "好的，我来帮你..." 这种敷衍回答)
        lazy_patterns = ["好的，我来帮你", "让我来回答", "当然可以",
                         "As an AI", "I apologize"]
        for pattern in lazy_patterns:
            if out.strip().startswith(pattern):
                issues.append(f"疑似敷衍回答: 以 '{pattern}' 开头")
                self.stats["lazy_answer"] += 1
                break

        # 4. 输入输出一致性(简单检查)
        if len(inp) > 0 and inp.strip() == out.strip():
            issues.append("input 和 output 完全相同")
            self.stats["identical_io"] += 1

        return len(issues) == 0, issues

    def clean_dataset(self, samples):
        """清洗数据集"""
        clean = []
        rejected = []

        for i, sample in enumerate(samples):
            ok, issues = self.check_sample(sample)
            if ok:
                # 去重(hash)
                clean.append(sample)
            else:
                rejected.append({"index": i, "issues": issues, "sample": sample})

        # 去重
        seen = set()
        deduped = []
        for s in clean:
            h = hashlib.md5(
                (s.get("instruction","") + s.get("output","")).encode()
            ).hexdigest()
            if h not in seen:
                seen.add(h)
                deduped.append(s)
            else:
                self.stats["duplicate"] += 1

        print(f"清洗结果: {len(samples)} → {len(deduped)} (通过率 {len(deduped)/max(len(samples),1)*100:.1f}%)")
        print(f"拒绝原因统计: {dict(self.stats)}")
        return deduped, rejected

# 演示
samples = [
    {"instruction": "解释什么是机器学习", "output": "机器学习是AI的一个分支...(详细回答)"},
    {"instruction": "hi", "output": "你好"},  # inst 太短
    {"instruction": "解释深度学习", "output": "好的，我来帮你解释深度学习..."},  # 敷衍回答
    {"instruction": "解释什么是机器学习", "output": "机器学习是AI的一个分支...(详细回答)"},  # 重复
    {"instruction": "对比CNN和RNN", "output": "CNN擅长...RNN适合..."},
]

cleaner = FinetuneDataCleaner()
clean, rejected = cleaner.clean_dataset(samples)

print(f"\n通过: {len(clean)} 条")
for s in clean:
    print(f"  ✅ {s['instruction'][:40]}...")

print(f"\n拒绝: {len(rejected)} 条")
for r in rejected:
    print(f"  ❌ [{r['index']}] {r['issues']}")
```

### 实战 2：数据多样性分析

```python
"""检查微调数据的多样性"""
from collections import Counter
import re

def analyze_diversity(samples):
    """分析数据集多样性"""
    stats = {}

    # 1. 指令长度分布
    inst_lens = [len(s.get("instruction", "")) for s in samples]
    stats["指令平均长度"] = sum(inst_lens) / max(len(inst_lens), 1)
    stats["指令长度范围"] = f"{min(inst_lens)}-{max(inst_lens)}"

    # 2. 任务类型分布(基于关键词)
    task_keywords = {
        "翻译": ["翻译", "translate"],
        "摘要": ["总结", "摘要", "概括", "summarize"],
        "问答": ["什么是", "如何", "为什么", "怎么"],
        "代码": ["写代码", "实现", "函数", "编程", "code"],
        "写作": ["写一篇", "撰写", "起草", "write"],
        "分析": ["分析", "对比", "比较", "分析"],
        "分类": ["分类", "归类", "classify"],
    }

    task_counts = Counter()
    for s in samples:
        text = s.get("instruction", "") + s.get("input", "")
        assigned = False
        for task, keywords in task_keywords.items():
            if any(kw in text.lower() for kw in keywords):
                task_counts[task] += 1
                assigned = True
                break
        if not assigned:
            task_counts["其他"] += 1

    stats["任务类型分布"] = dict(task_counts)

    # 3. 输出长度分布
    out_lens = [len(s.get("output", "")) for s in samples]
    bins = [0, 50, 100, 200, 500, 1000, 2000, 99999]
    bin_labels = ["0-50", "50-100", "100-200", "200-500", "500-1000", "1000-2000", "2000+"]
    hist = [0] * len(bin_labels)
    for l in out_lens:
        for i in range(len(bins)-1):
            if bins[i] <= l < bins[i+1]:
                hist[i] += 1
                break
    stats["输出长度分布"] = dict(zip(bin_labels, hist))

    return stats

# 演示
print("=== 数据多样性分析 ===\n")
demo_stats = analyze_diversity(samples)
for k, v in demo_stats.items():
    print(f"{k}:")
    if isinstance(v, dict):
        for sub_k, sub_v in v.items():
            print(f"  {sub_k}: {sub_v}")
    else:
        print(f"  {v}")
    print()
```

### 实战 3：LLM-as-Judge 自动评估

```python
"""用 LLM 自动评估微调前后效果"""
print("""
# === LLM-as-Judge 评估示例 ===
from openai import OpenAI

client = OpenAI(
    api_key="your-api-key",
    base_url="https://api.deepseek.com",
)

JUDGE_PROMPT = '''你是一个AI效果评估专家。请对以下两个回答进行评分。
评分维度(每项1-5分):
- 准确性: 回答是否事实正确
- 完整性: 是否覆盖问题的所有方面
- 流畅性: 语言是否自然流畅
- 格式: 结构化程度和可读性

问题: {question}

回答A(微调前): {answer_a}
回答B(微调后): {answer_b}

请给出各维度的评分和简要理由，格式为JSON:
{{
    "A_scores": {{"accuracy": X, "completeness": X, ...}},
    "B_scores": {{"accuracy": X, "completeness": X, ...}},
    "winner": "A" 或 "B",
    "reason": "..."
}}
'''

def judge(question, answer_a, answer_b):
    prompt = JUDGE_PROMPT.format(
        question=question, answer_a=answer_a, answer_b=answer_b
    )
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.1,
    )
    return response.choices[0].message.content

# 使用
# result = judge(
#     "解释什么是机器学习",
#     "ML is a field of AI...",  # 微调前
#     "机器学习(Machine Learning)是人工智能的一个分支...(结构化回答)",  # 微调后
# )
# print(result)
''')
print("LLM-as-Judge 评估的优势:")
print("  1. 比 BLEU/ROUGE 更准确(理解语义而非匹配 n-gram)")
print("  2. 可自定义评分维度")
print("  3. 成本低(~0.01元/次评估)")
print("\n注意事项:")
print("  - 用不同的 LLM 做评估(避免自己评自己)")
print("  - 多次评估取平均(单次评估有随机性)")
print("  - 人工抽查 10-20% 的结果校准")

# 模拟评估结果
demo_result = {
    "A_scores": {"accuracy": 3, "completeness": 2, "fluency": 4, "format": 2},
    "B_scores": {"accuracy": 4, "completeness": 4, "fluency": 4, "format": 5},
    "winner": "B",
    "reason": "B 回答更完整、格式更清晰、更适合学习场景"
}
print(f"\n模拟评估结果:\n{json.dumps(demo_result, ensure_ascii=False, indent=2)}")
```

## 进阶方向

### 数据工程的常见陷阱

| 陷阱 | 表现 | 规避方法 |
|------|------|---------|
| **数据泄露** | 测试集样本出现在训练集中 | 严格的数据隔离，去重检查跨训练/测试集 |
| **标签偏差** | 标注员有系统性偏好 | 多标注员交叉验证，计算标注一致性 |
| **长尾缺失** | 数据集中在热门场景，冷门场景覆盖不足 | 主动采样，按主题均衡配比 |
| **模板化** | LLM 生成的回答全是同一个套路 | 在生成 Prompt 中加多样性约束 |

### 人工评估的必要性

尽管自动化评估很方便，但以下情况必须人工参与：

1. **新领域首次评估**：没有参考标准时，人判断"对不对"
2. **安全相关任务**：医疗、法律场景的输出必须专家审核
3. **创意性任务**：文章的"好不好"很难用自动化指标衡量
4. **校准自动评估**：定期抽查 10-20%，校准 LLM Judge 的准确率

## 常见问题

### 小白最常踩的 3 个坑

1. **数据量和质量的关系搞反**
   - 错误："数据越多越好，10 万条总比 1 万条强"
   - 正确：5000 条高质量 > 10 万条低质量。如果 10 万条中有 80% 存在格式错误和敷衍回答，模型会学会"敷衍"而不是"精确"

2. **评估集和训练集来自同一来源**
   - 错误：训练集和测试集都是同一个 GPT-4 批量生成的（模板相同）
   - 正确：测试集应该独立于训练数据来源（不同 LLM 生成、不同场景、不同风格），才能测出真实泛化能力

3. **忽略 SFT 数据的指令多样性**
   - 错误：所有数据都是"解释XX"格式
   - 正确：覆盖多种指令类型——问答、生成、翻译、总结、改写、推理、代码等。指令多样性 > 内容多样性

### 自检题

**Q1**：为什么 RLHF/DPO 数据格式是 "chosen/rejected 对" 而不是单独的 "好回答"？

> **答案**：RLHF 的目标不是教会模型"什么是标准答案"，而是教会模型"什么是更好的回答"。用对比对（A 比 B 好）训练，模型学的是相对偏好而非绝对标准。这更符合实际场景——很多问题没有唯一正解，但有"更好"和"更差"之分。

**Q2**：LLM 生成微调数据的最大风险是什么？如何缓解？

> **答案**：最大风险是模型学会复制 LLM 的回答模式（包括其错误和偏差），而非真正学会任务能力——这被称为"模型坍缩"。缓解方法：(1) 用多个不同 LLM 生成数据；(2) 人工审核至少 10%；(3) 数据生成时加多样性约束 Prompt；(4) 混入人工撰写的数据。

**Q3**：评估微调效果时，只看"测试集分数"够吗？还需要看什么？

> **答案**：不够。还需要看：(1) 退化检查——微调后的模型是否在原本擅长的通用任务上变差了（"灾难性遗忘"）；(2) 过拟合检查——训练集和测试集表现差距是否过大；(3) 真实场景表现——测试集可能和实际用户行为有差距，需要线上 A/B 测试。
