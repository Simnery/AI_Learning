## 核心概念

### 什么是知识蒸馏

**一句话**：知识蒸馏是让一个"大模型"（教师）把自己的能力教给一个"小模型"（学生），让小模型用更少的参数和计算量达到接近大模型的效果。

```
教师模型(大):  DeepSeek-V3 671B → 推理成本高、部署困难
                 ↓ 蒸馏
学生模型(小):  Qwen2.5-7B      → 推理快、成本低、单卡可跑
```

**类比**：大学教授（教师模型）用多年经验写一本精华教材（蒸馏数据），本科生（学生模型）读完教材就能掌握核心知识——不需要再经历教授几十年研究过程。

### 为什么要蒸馏

| 场景 | 教师模型 | 学生模型 | 价值 |
|------|---------|---------|------|
| 端侧部署 | GPT-4 | 1.5B 小模型 | 手机/嵌入式设备可运行 |
| 成本优化 | DeepSeek-V3 | Qwen2.5-7B | API 成本降 10-100 倍 |
| 延迟优化 | 大模型(500ms) | 小模型(20ms) | 实时交互场景 |
| 领域专精 | 通用大模型 | 垂直小模型 | 特定领域效果不降反升 |

## 原理讲解

### 知识蒸馏的核心思想

传统训练和蒸馏的本质区别：

```
传统训练: 模型学习"标准答案"（硬标签）
  输入: "1+1=?"
  → 标准答案: "2"
  → 模型只学到一个信息: 答案是 2

蒸馏训练: 模型学习"老师的思考过程"（软标签）
  输入: "1+1=?"
  → 老师的输出概率分布:
    "2": 99.9%
    "3": 0.05%
    "1": 0.03%
    "0": 0.02%
  → 模型学到: 正确答案是 2，但它还知道"3"比"0"更接近正确答案
```

**软标签的价值**：教师模型的概率分布包含了丰富的"暗知识"——不仅告诉学生"什么是对的"，还告诉学生"错有多错"。2 当然比 3 好，但 3 比 99 好得多——这些相对关系是传统训练无法提供的。

### 经典蒸馏方法对比

| 方法 | 怎么做 | 优点 | 缺点 |
|------|--------|------|------|
| **Logits 蒸馏** | 学生模仿教师的输出概率分布 | 信息量大 | 需要访问教师 logits |
| **特征蒸馏** | 学生模仿教师中间层的隐藏状态 | 更深层的知识迁移 | 架构需对齐 |
| **数据蒸馏（推荐）** | 教师生成高质量训练数据，学生用这些数据做 SFT | 最简单，不需要教师参与蒸馏过程 | 丢失软标签信息 |

**LLM 时代推荐**：数据蒸馏——教师生成数据 → 学生 SFT 微调。无需同时加载教师和学生，支持不同的模型架构。

### Unsloth：微调加速利器

Unsloth 是目前最快的 LLM 微调框架之一：

```
传统微调: HuggingFace Trainer → 7B 模型 LoRA 需 16GB，训练速度 1x
Unsloth:  手写 CUDA 内核优化   → 7B 模型 LoRA 需 8GB，训练速度 2-5x

核心优化:
  - 重写注意力计算内核(Flash Attention 风格)
  - 在线反量化(减少显存搬运)
  - 自动混合精度 + 梯度检查点
```

### 蒸馏效果评估

| 评估维度 | 方法 | 目标 |
|---------|------|------|
| **能力保留度** | 学生在原任务上的表现 vs 教师 | >80% 教师水平 |
| **推理速度** | Token/s 对比 | >5x 教师速度 |
| **显存占用** | 部署所需 GPU 显存 | <50% 教师显存 |
| **领域深度** | 在专业领域的表现 | 可不低于教师 |

### 实战 1：数据蒸馏——教师模型生成训练数据

```python
"""数据蒸馏: 用大模型(教师)生成高质量训练数据，用于训练小模型(学生)"""
from openai import OpenAI
import json

# 教师模型: DeepSeek-V3（大模型）
teacher = OpenAI(
    api_key="your-deepseek-key",
    base_url="https://api.deepseek.com",
)

def teacher_generate(instruction, input_text=""):
    """教师模型生成高质量回答"""
    response = teacher.chat.completions.create(
        model="deepseek-chat",  # 教师模型
        messages=[
            {"role": "system", "content": "你是一个AI技术专家，回答要详细、准确、结构化。"},
            {"role": "user", "content": f"{instruction}\n{input_text}"},
        ],
        temperature=0.3,  # 低温度保证质量
        max_tokens=1000,
    )
    return response.choices[0].message.content

# 蒸馏数据模板
distillation_templates = [
    {"instruction": "解释以下AI概念，并给出一个简单例子", "input": "注意力机制"},
    {"instruction": "对比以下两种技术，列出关键区别", "input": "LoRA vs 全量微调"},
    {"instruction": "解释以下概念，要求用大白话让人听懂", "input": "Transformer"},
    {"instruction": "回答以下问题并给出代码示例", "input": "如何在Python中使用FAISS做向量检索"},
]

print("=== 数据蒸馏：教师模型生成训练数据 ===\n")

distillation_data = []
for i, template in enumerate(distillation_templates):
    print(f"教师生成 {i+1}/{len(distillation_templates)}: {template['instruction'][:30]}...")
    # 实际调用 API
    # output = teacher_generate(template["instruction"], template["input"])

    # 演示: 模拟教师输出
    output = f"[教师生成] 对 '{template['input']}' 的详细回答..."

    distillation_data.append({
        "instruction": template["instruction"],
        "input": template["input"],
        "output": output,
    })

# 保存蒸馏数据，用于后续训练学生模型
with open("distillation_data.json", "w", encoding="utf-8") as f:
    json.dump(distillation_data, f, ensure_ascii=False, indent=2)

print(f"\n蒸馏数据已保存: {len(distillation_data)} 条")
print("下一步: 用这些数据微调小模型(学生)，使其接近教师水平")
```

### 实战 2：Unsloth LoRA 微调实操

```python
"""用 Unsloth 快速微调（代码框架）"""
print("""
# === Unsloth 微调完整流程 ===

from unsloth import FastLanguageModel
from unsloth import is_bfloat16_supported
import torch
from datasets import load_dataset

# ===== 1. 加载模型 =====
MODEL_NAME = "unsloth/Qwen2.5-7B"  # 学生模型

model, tokenizer = FastLanguageModel.from_pretrained(
    model_name=MODEL_NAME,
    max_seq_length=4096,       # 上下文长度
    dtype=None,                # None = 自动检测
    load_in_4bit=True,         # 4-bit 量化加载，节省显存
)

# ===== 2. 添加 LoRA =====
model = FastLanguageModel.get_peft_model(
    model,
    r=16,                      # LoRA 秩
    target_modules=[           # 哪些层加 LoRA
        "q_proj", "k_proj", "v_proj", "o_proj",
        "gate_proj", "up_proj", "down_proj",
    ],
    lora_alpha=16,
    lora_dropout=0.1,
    use_gradient_checkpointing="unsloth",  # Unsloth 优化的梯度检查点
)

# ===== 3. 格式化数据 =====
def format_prompt(example):
    '''根据任务格式化每条数据'''
    return {
        "text": (
            f"<|im_start|>system\\n你是AI助教,回答要准确详细<|im_end|>\\n"
            f"<|im_start|>user\\n{example['instruction']}:{example['input']}<|im_end|>\\n"
            f"<|im_start|>assistant\\n{example['output']}<|im_end|>"
        )
    }

# 加载蒸馏数据
dataset = load_dataset("json", data_files="distillation_data.json", split="train")
dataset = dataset.map(format_prompt)

# ===== 4. 训练 =====
from trl import SFTTrainer
from transformers import TrainingArguments

trainer = SFTTrainer(
    model=model,
    tokenizer=tokenizer,
    train_dataset=dataset,
    dataset_text_field="text",
    max_seq_length=4096,
    args=TrainingArguments(
        per_device_train_batch_size=2,
        gradient_accumulation_steps=4,    # 等效 batch_size=8
        warmup_steps=10,
        max_steps=200,                    # 小数据集训练步数
        learning_rate=2e-4,
        fp16=not is_bfloat16_supported(),
        bf16=is_bfloat16_supported(),
        logging_steps=10,
        output_dir="./unsloth_output",
        seed=42,
    ),
)

print("开始训练...")
trainer_stats = trainer.train()

# ===== 5. 保存模型 =====
# 方案A: 保存 LoRA 适配器(推荐,文件小)
model.save_pretrained("./lora_adapter")
tokenizer.save_pretrained("./lora_adapter")

# 方案B: 合并并保存完整模型(文件大,但可直接部署)
# model.save_pretrained_merged("./full_model", tokenizer, save_method="merged_16bit")

print(f"训练完成! 用时: {trainer_stats.metrics['train_runtime']:.0f}秒")

# ===== 6. 推理测试 =====
FastLanguageModel.for_inference(model)

def test_model(instruction, input_text=""):
    prompt = f"<|im_start|>user\\n{instruction}:{input_text}<|im_end|>\\n<|im_start|>assistant\\n"
    inputs = tokenizer([prompt], return_tensors="pt").to(model.device)
    outputs = model.generate(**inputs, max_new_tokens=256, temperature=0.7)
    return tokenizer.decode(outputs[0], skip_special_tokens=True)

print("\\n=== 测试蒸馏效果 ===")
print(test_model("解释什么是RAG技术"))
''')
print("Unsloth 微调流程说明:")
print("  1. 加载模型(4-bit 量化) → 2. 添加 LoRA(仅训练 0.5-1% 参数)")
print("  3. 格式化数据 → 4. 训练(200 steps)")
print("  5. 保存适配器 → 6. 测试推理")
print("\n硬件需求: 7B 模型仅需 8-10GB 显存(单张 4060/4070/4090)")
```

## 进阶方向

### LLM 蒸馏的挑战

| 挑战 | 说明 | 缓解方案 |
|------|------|---------|
| **能力分布不均** | 蒸馏后某些能力衰退严重 | 多任务评估，薄弱能力追加数据 |
| **教师偏差传染** | 学生学会了教师的系统偏差 | 多教师蒸馏，混合不同教师的数据 |
| **评估不完整** | 只看几个指标，漏掉退化维度 | 全维度评估，含通用能力测试 |
| **放大幻觉** | 学生放大教师的幻觉倾向 | 数据中加入"不确定"、"拒绝回答"样本 |

### 蒸馏策略选择

```
场景1: API 成本优化
  教师: GPT-4/DeepSeek API
  学生: Qwen2.5-7B 开源模型
  策略: 数据蒸馏 → SFT
  效果: API 成本降 95%，效果保留 85%+

场景2: 端侧部署
  教师: Qwen2.5-72B
  学生: Qwen2.5-1.5B
  策略: Logits 蒸馏 + 数据蒸馏
  效果: 手机可跑 1.5B 模型，特定任务接近 72B

场景3: 领域特化
  教师: DeepSeek-V3
  学生: Llama-3-8B
  策略: 领域数据蒸馏 + LoRA
  效果: 法律/医疗特定领域可能超越通用大模型
```

## 常见问题

### 小白最常踩的 3 个坑

1. **蒸馏数据全是同一模板**
   - 错误：教师用相同格式生成 10000 条数据，学生变成了"格式复制机"
   - 正确：在生成 Prompt 中加入格式多样性指令，或分多批用不同 Prompt 生成

2. **不评估蒸馏后通用能力是否下降**
   - 错误：只看领域任务评估（法律写作准确率从 50%→85%），没发现基础推理能力下降了 30%
   - 正确：蒸馏后必须同时评估通用能力（MMLU、GSM8K 等标准测试），确保"灾难性遗忘"可控

3. **Unsloth 加载模型后忘记切换到推理模式**
   - 错误：`trainer.train()` 完后直接用 model 推理，显存爆炸
   - 正确：推理前调用 `FastLanguageModel.for_inference(model)`

### 自检题

**Q1**：数据蒸馏和 Logits 蒸馏的核心区别是什么？为什么 LLM 时代更推荐数据蒸馏？

> **答案**：数据蒸馏是教师生成数据→学生用数据微调，无需同时加载教师和学生，支持不同架构。Logits 蒸馏需要同时运行教师和学生以获取概率分布，LLM 时代因为教师模型太大（671B），同时加载两个模型显存不现实，所以数据蒸馏更实用。另外 GPT-4 等闭源模型不暴露 logits，只能用数据蒸馏。

**Q2**：蒸馏后的学生模型有没有可能在某些方面超越教师模型？什么情况下会发生？

> **答案**：有可能。当学生模型通过蒸馏聚焦于某一特定领域时，蒸馏数据经过精心筛选和优化，质量可能高于教师训练数据的平均水平。另外，蒸馏时可以对领域数据进行过采样，学生在特定领域的"深度"可能超过通用大模型。这就像"专科医生"可能在某个病种上比"全科医生"更强。

**Q3**：Unsloth 为什么比标准 HuggingFace Trainer 快？核心优化是什么？

> **答案**：核心是手写 CUDA 内核优化：1) 重写了注意力计算（类似 Flash Attention），减少显存 IO；2) 在线反量化（在计算时即时转换而非先全部加载到显存再转）；3) 算子融合（将多个小 CUDA 算子合并为一个，减少 kernel launch 开销）；4) 针对 LoRA 的特定优化（只计算 LoRA 层的梯度，跳过冻结参数的梯度计算）。
