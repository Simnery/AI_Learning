## 原理讲解

微调面试的三层能力验证：
- **概念层**：预训练 vs 微调 vs PEFT vs LoRA 的定义和区别（书本知识）
- **工程层**：显存怎么估算、数据怎么清洗、过拟合怎么检测、训练 loss 不下降怎么办（工程经验）
- **策略层**：什么时候该微调、什么时候不该微调、微调后的模型如何评估和上线（架构决策）

面试官通过"显存计算题"（7B LoRA 需要多少显存？）和"故障诊断题"（训练 loss 震荡怎么办？）快速判断你的工程深度。

### 4. 微调 vs RAG 面试话术

| 问法 | 要点 |
|------|------|
| 何时微调 | 风格/格式固定、领域术语密集、RAG 上下文不够 |
| 何时 RAG | 知识更新频、需引用溯源、无训练算力 |
| 组合 | RAG 检索 + 微调模型生成，各管知识与口吻 |

## 代码实战

```python
# pip install torch

"""LoRA 微调核心代码 —— 面试中展示微调原理"""

import torch
import torch.nn as nn

class LoRALinear(nn.Module):
    """LoRA 的核心实现 —— 面试中能写出这个很加分
    
    原理: h = Wx + BAx
    - W 是原始权重（冻结，不训练）
    - A 和 B 是低秩矩阵（只训练这两个，参数量极小）
    """
    def __init__(self, in_features: int, out_features: int, r: int = 16, alpha: float = 16):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features, bias=False)
        self.linear.weight.requires_grad = False  # 冻结原始权重
        
        # LoRA 低秩矩阵
        self.lora_A = nn.Parameter(torch.zeros(in_features, r))
        self.lora_B = nn.Parameter(torch.zeros(r, out_features))
        self.scaling = alpha / r
        
        nn.init.kaiming_uniform_(self.lora_A, a=5**0.5)
        nn.init.zeros_(self.lora_B)
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        frozen_out = self.linear(x)
        lora_out = (x @ self.lora_A) @ self.lora_B * self.scaling
        return frozen_out + lora_out


# ===== 显存估算工具（面试常见计算题） =====
def estimate_memory(model_params_b: float, lora_rank: int = 16, 
                    precision: str = "fp16") -> dict:
    """估算 LoRA 微调显存需求
    
    面试常考：7B 模型 LoRA r=16 需要多少显存？
    """
    bytes_per_param = 2 if precision == "fp16" else 4
    
    base_memory = model_params_b * bytes_per_param  # 原始权重
    lora_params = model_params_b * 0.001  # LoRA 参数 ≈ 0.1% 原始
    lora_memory = lora_params * bytes_per_param
    grad_memory = lora_params * bytes_per_param * 3  # 梯度+优化器
    
    return {
        "base_weights_gb": round(base_memory, 1),
        "lora_weights_gb": round(lora_memory, 1),
        "gradients_gb": round(grad_memory, 1),
        "total_gb": round(base_memory + lora_memory + grad_memory, 1),
    }


# 演示：7B 模型 LoRA 显存估算
mem = estimate_memory(7.0, lora_rank=16)
print(f"7B LoRA: 总显存 {mem['total_gb']}GB (原始 {mem['base_weights_gb']}GB + LoRA {mem['lora_weights_gb']}GB)")

# 输出: 7B LoRA: 总显存 ~14.3GB (原始 14.0GB + LoRA ~0.1GB)
```

## 常见问题

### 面试题与回答要点（精选）

### Q1: 预训练和微调的理解

**预训练**：在海量通用数据上从头训练模型（学语言基本能力，成本千万级）。**微调**：在预训练模型基础上用特定领域数据小幅度更新参数（学特定技能，成本百-万元级）。类比：预训练=读完整个图书馆，微调=学一门专业课。

### Q2-Q3: 全参数微调 vs PEFT

- **全参数微调**：更新所有参数。优点：表现最好。缺点：显存大（7B→56GB）、慢、灾难性遗忘
- **PEFT**：只训练少量新增参数。优点：显存小（7B→6GB with QLoRA）、快、保留原有能力。缺点：表现略低于全量（差距在缩小）

### Q4: LoRA原理详解

**核心**：在原始权重W旁加两个低秩小矩阵A和B，`h=Wx+BAx`。W冻结，只训练A和B。秩r=16时参数量仅为原始的0.1%-1%。QLoRA在此基础上将W量化为4-bit（NormalFloat4），进一步降低显存。

### Q5: 微调技术选型

**决策树**：1）资源：GPU显存？<16GB→QLoRA，>40GB→LoRA或全量 2）数据量：<1000条→LoRA（全量会过拟合），>10000条→可考虑全量 3）目标：加新能力→LoRA，改变风格→全量或高rank LoRA。

### Q6: 微调数据来源

三种来源：1）开源数据集（HuggingFace/ModelScope，关键词+质量筛选）2）业务数据（用户反馈/人工标注，最贵但最精准）3）合成数据（用强模型生成→弱模型学习，量大但可能有偏差）。我的项目三者比例约1:3:6。

### Q7: 高质量微调数据的标准

**五条标准**：1）多样性（覆盖不同场景）2）准确性（事实正确，标注无误）3）一致性（格式统一，同一问题的答案一致）4）代表性（和线上真实请求分布一致）5）平衡性（各任务类型比例均衡）。

### Q8: SFT指令微调数据格式

```json
{"messages": [
  {"role": "system", "content": "你是XX助手"},
  {"role": "user", "content": "问题"},
  {"role": "assistant", "content": "期望回答"}
]}
```
关键：system定义角色行为、user指令清晰具体、assistant回答是训练目标。

### Q9-Q10: 微调效果评估

**自动化评估**：BLEU/ROUGE（生成质量）、困惑度PPL（语言流畅度）、Benchmark（MMLU/C-Eval知识能力）。**人工评估**：盲评A/B测试（不知道哪个是微调版）、多维度打分（准确性/流畅性/安全性）。

我倾向：自动化做初筛（快速定位差模型），人工做终评（真正判断好坏）。

### Q11: 模型蒸馏核心思想

Teacher-Student：用大模型（Teacher）的输出（软标签/推理过程）训练小模型（Student）。软标签含丰富信息（不只正确答案，还有"其他选项有多接近"），让小模型学到大模型的推理能力。温度参数控制软标签的平滑度。

### Q12-Q13: 多模态与CV三大任务

- **三大任务**：分类（整图→标签）、检测（定位+分类）、分割（逐像素分类）
- **VQA vs 传统CV**：VQA输入是图片+自然语言问题→输出自然语言答案。传统CV输出是固定类别的标签/框/掩码。VQA更通用但精度不如专用CV模型

### Q14: YOLO vs Qwen-VL 质检选型

**YOLO**：需要标注但精度高（mAP>0.9）、实时（<10ms）、可边缘部署。**Qwen-VL**：零样本、能描述缺陷原因、灵活但慢（秒级）。选择：缺陷类型固定+有标注→YOLO；类型多变+需原因分析→Qwen-VL；预算允许→YOLO初筛+Qwen-VL复核。


### 面试常见误区

**误区 1：把"调过 API"说成"做过微调"**

面试官问"你做过模型微调吗"，回答"我用过 GPT-4 API"——这不叫微调。只有真实改过模型权重（用 LoRA/QLoRA/全量微调）才算。如果你真的没做过，坦诚说"我理解 LoRA 原理但还没实操过"。

**误区 2：不准备显存计算题**

面试官可能现场出题："7B 模型 LoRA r=16，需要多少显存？"提前记住公式：FP16 权重=7×2=14GB，LoRA 参数≈7B×0.1%=7MB，梯度≈LoRA参数×3=21MB，总约 14.3GB。展示你的工程思维。

**误区 3：数据质量回答太空洞**

"微调数据要高质量"等于没说。要具体："我们用了三层过滤——格式校验（JSON合法）→规则过滤（长度/语言/重复度）→LLM评分（1-5分，<3分丢弃），最终保留率约 40%。"
