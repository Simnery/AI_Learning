## 2.1 机器学习范式

**核心目标**：理解三种学习范式的区别，知道什么时候用哪一种。

| 子主题 | 你必须搞懂的内容 | 时间 |
|--------|------------------|------|
| 监督学习 (Supervised) | 有标签数据 (X→Y)，分类 vs 回归 | 1 天 |
| 无监督学习 (Unsupervised) | 无标签数据，聚类、降维 | 1 天 |
| 强化学习 (Reinforcement) | Agent 与环境交互，奖励驱动 | 1 天 |

**BIOS 类比**：
- 监督学习 → 有 JEDEC/PCIe Spec 的开发（知道正确答案）
- 无监督学习 → 逆向分析未知 BIOS module（没有 spec，自己找规律）
- 强化学习 → 盲调电压到稳定（试错，看结果调整）

## 2.2 核心概念

**核心目标**：掌握机器学习项目的标准工作流——训练、评估、避免过拟合。

| 子主题 | 你必须搞懂的内容 | 时间 |
|--------|------------------|------|
| 损失函数 (Loss Function) | MSE、交叉熵的区别与适用场景 | 1 天 |
| 梯度下降 | 批量 vs 随机 vs Mini-batch 的权衡 | 1 天 |
| 过拟合 vs 欠拟合 | 偏差-方差权衡，如何诊断 | 1 天 |
| 训练/验证/测试集 | 为什么必须分三份，怎么分 | 0.5 天 |
| 正则化 | L1/L2、Dropout——防过拟合的武器 | 0.5 天 |

**BIOS 类比**：
- Loss → POST Code 报错，告诉你当前状态离"能开机"差多远
- 过拟合 → 代码只在自家 CRB 上跑过，量产到 ODM 板子全挂
- 训练/验证/测试 → 开发调测 (bring-up) / QA 验证 / 客户现场验收
- 正则化 → 写代码加 assert 和边界检查，防止路径上特殊 case 崩溃

## 2.3 经典算法速览

**核心目标**：不需要手写实现，但要知道有哪些"工具"，能对号入座。

| 算法 | 类型 | BIOS 类比 | 时间 |
|------|------|-----------|------|
| 线性回归 | 回归 | 电压-频率曲线拟合 | 0.5 天 |
| 逻辑回归 | 分类 | POST Code 判断能否进系统 | 0.5 天 |

## 2.4 从理论到代码：第一个 ML 项目

**核心目标**：用 PyTorch 跑通 MLP + MNIST 手写数字分类。这是你第一个完整的 pipeline。

| 步骤 | 内容 | 时间 |
|------|------|------|
| 加载数据 | PyTorch DataLoader，MNIST 数据集 | 0.5 天 |
| 定义模型 | 3 层 MLP (784 → 128 → 64 → 10) | 0.5 天 |
| 训练循环 | forward → loss → backward → optimizer.step() | 1 天 |
| 评估 | 验证集 accuracy，混淆矩阵 | 0.5 天 |

**这个项目的价值**：你将在 03_深度学习基础 中画这 4 个步骤的图并理解"为什么"，而不是"是什么"。
